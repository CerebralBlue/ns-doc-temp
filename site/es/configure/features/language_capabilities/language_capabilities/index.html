    Capacidades del lenguaje

    Identificar idioma

    ¿Qué es?

    • NeuralSeek proporciona un servicio que analizaría e identificaría el idioma de un texto dado.

    ¿Por qué es importante?

    • Cualquier aplicación que necesite entender en qué idioma está escrito un texto determinado puede usar ahora NeuralSeek para hacerlo, en lugar de depender de otros servicios externos.

    ¿Cómo funciona?

    • La identificación de idiomas se proporciona como API REST y se puede probar en la documentación de la API de NeuralSeek. El mensaje de carga útil está en formato text/plain y contiene text en ciertos idiomas. Un mensaje de ejemplo se vería así:
    ¿En qué idioma está este texto?
    • NeuralSeek entonces identificaría en qué idioma está escrito esto y devolvería el código de idioma y el puntaje de confianza:
    [
    {
        language: es,
        confidence: 0.95
    }
]

    Categorización de intenciones

    ¿Qué es?

    • NeuralSeek puede categorizar automáticamente las entradas y preguntas de los usuarios en categorías. Estas categorías pueden ser de cualquier tipo: productos, organizaciones, departamentos, etc. Los usuarios pueden configurar categorías en la pestaña Configurar, ingresando nombres y descripciones de categorías. Estos se utilizarán entonces para hacer coincidir las entradas de los usuarios con las categorías. Las entradas de los usuarios que no coincidan con ninguna categoría, o que coincidan demasiado estrechamente con varias categorías, se colocarán en una categoría predeterminada llamada Otro. Esta categoría predeterminada no se puede modificar.

    ¿Por qué es importante?

    • La categorización es muy útil para escalar NeuralSeek dentro de una organización. Al agrupar las intenciones en categorías, puede facilitar mucho que los expertos en la materia tomen medidas rápidamente en su área específica de contenido. La categorización puede ser útil incluso fuera del contexto de responder preguntas de los usuarios, por ejemplo, en el enrutamiento de preguntas de los clientes al departamento o agente en vivo correcto. La categorización se puede llamar directamente a través de la API.

    ¿Cómo funciona?

    • La entrada del usuario se puntúa y se agrupa en función del título y la descripción de la categoría, y en función de las intenciones que se han movido manualmente a las categorías (aprendizaje automático). Una vez que se habilita la categorización, las pantallas Curar y Analítica cambiarán para mostrar agrupaciones en torno a las categorías. La categorización no es retroactiva, lo que significa que si defines una nueva categoría, no volveremos a ejecutar automáticamente todas las entradas de usuario anteriores en las nuevas categorías. Los usuarios pueden mover manualmente las intenciones a las categorías a través de la pestaña Curar o las funciones de descarga/edición de CSV. Las ediciones realizadas se utilizarán para entrenar al sistema para futuros eventos de categorización.

    Traducción de idiomas

    ¿Qué es?

    • NeuralSeek proporciona traducción de idiomas que permitirá a los usuarios llamarlo para traducir idiomas a diferentes idiomas.

    ¿Por qué es importante?

    • Cualquier aplicación que necesite traducir un texto dado a otro idioma puede usar ahora NeuralSeek para hacerlo, en lugar de depender de otros servicios de traducción externos.

    ¿Cómo funciona?

    • La traducción se proporciona como API REST y se puede probar en la documentación de la API de NeuralSeek.
    • El mensaje de carga útil está en formato JSON y contiene una matriz de text en cierto(s) idioma(s). Otro atributo es target que especifica el idioma de destino en el que se debe realizar la traducción. Un mensaje de ejemplo se vería así:
    {
    text: [
    NeuralSeek introdujo varios nuevos recursos en julio de 2023, incluidas respuestas en streaming para casos de uso web, soporte multilingüe mejorado, curar a CSV/cargar QA curada desde CSV, análisis de coincidencia semántica mejorado, compatibilidad con el modelo IBM WatsonX actualizado y monitoreo de ida y vuelta de AWS Lex.
    ],
    target: es
},

    Para obtener más detalles sobre qué códigos de idioma se admiten, consulte Compatibilidad con varios idiomas. (es) en la solicitud al invocar Seek.

    La misma se puede lograr al invocar Seek utilizando la API REST. Puede especificar el idioma en options > language.

    Soporte multilingüe para KBs

    NeuralSeek ofrece un sólido soporte multilingüe, lo que permite a los usuarios interactuar con una base de conocimientos (KB) en un idioma diferente al que está escrita. Esto es particularmente útil en escenarios donde la base de conocimientos está en un idioma (por ejemplo, inglés), pero los usuarios necesitan consultarla en otro idioma (por ejemplo, español).

    Cómo funciona

    Cuando un usuario consulta la base de conocimientos en un idioma diferente, NeuralSeek maneja el proceso de traducción de manera transparente:

    1. Consulta del usuario en su idioma nativo: El usuario hace una pregunta en su idioma nativo (por ejemplo, español).
    2. Traducción al idioma de la KB: NeuralSeek traduce la pregunta del usuario al idioma de la base de conocimientos (por ejemplo, inglés).
    3. Consulta de la KB: La pregunta traducida se utiliza para buscar en la base de conocimientos.
    4. Recuperación de la respuesta: NeuralSeek recupera la respuesta del LLM en el idioma nativo del usuario.
    5. Entrega de la respuesta: El usuario recibe la respuesta en su idioma nativo.

    Ejemplo de escenario

    Pregunta en español, KB en inglés

    1. Consulta del usuario: ¿Cuál es la capital de Francia?
    2. Traducir al inglés: What is the capital of France?
    3. Consultar la KB en inglés: El sistema busca "What is the capital of France?" en la base de conocimientos en inglés.
    4. Recuperar la respuesta del LLM en español: La capital de Francia es París.
    5. Entregar la respuesta: La capital de Francia es París.
