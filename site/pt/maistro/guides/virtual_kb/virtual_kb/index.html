    Conhecimento Virtual

    Visão Geral

    O que é?

    • O KB Virtual é um recurso no mAIstro que permite que você defina um fluxo e o use como uma base de conhecimento virtual. Este recurso permite que você combine várias fontes de conhecimento em uma única base de conhecimento unificada, fornecendo uma solução mais abrangente e flexível para suas necessidades de recuperação de informações.

    Por que é importante?

    • Um KB Virtual melhora a pesquisa e descoberta do seu aplicativo, integrando várias fontes de conhecimento, entregando resultados mais abrangentes e relevantes. Ele oferece flexibilidade e escalabilidade, permitindo que você ajuste facilmente as fontes de conhecimento à medida que suas necessidades mudam.

    Como funciona?

    • O KB Virtual permite que você conecte e integre várias fontes de conhecimento, como bancos de dados, sistemas de gerenciamento de conteúdo e APIs externas, em uma única base de conhecimento virtual. Comece construindo um fluxo no mAIstro utilizando nossa variedade de funções e conectores nativos ou consulte nosso modelo de exemplo do KB Virtual para um guia fácil sobre como configurar um KB Virtual.

    Modelo de Exemplo no mAIstro

    1. Navegue até a guia mAIstro em sua instância do NeuralSeek.
    2. Clique em Modelos de Exemplo e procure o modelo intitulado Virtual KB.

    Este fluxo utiliza os nós Virtual In e Virtual Out, localizados abaixo de RAG Tools no menu lateral. Ele passa um conector de Pesquisa DuckDuckGo e um conector de API REST com uma URL da Wikipédia para o Modelo de Linguagem Grande para geração de respostas na guia Seek. Agora podemos utilizar a World Wide Web como uma fonte de conhecimento para a geração de respostas.

    Selecionando um KB Virtual

    1. Navegue até a guia Configurar em sua instância do NeuralSeek.
    2. Expanda o acordeão Conexão da Base de Conhecimento.
    3. Para o Tipo de Base de Conhecimento, selecione a opção KB Virtual.
    4. Para o modelo de KB Virtual do mAIstro, selecione a opção ex_Virtual_KB.
    5. Clique no ícone vermelho Salvar na parte inferior da tela para salvar sua configuração.

    Procurar com um Teclado Virtual

    1. Navegue até a guia Procurar na sua instância NeuralSeek.
    2. Digite qualquer pergunta. Por exemplo, Quem é Taylor Swift?
    3. Clique no botão Procurar para gerar uma resposta.

    Ao analisarmos a resposta gerada, podemos destacar os detalhes estatísticos e a fonte trazida de volta pelo NeuralSeek. A resposta é sintetizada a partir de uma combinação de pesquisas do DuckDuckGo e da Wikipédia relacionadas à cantora. Nossa análise semântica nos informa sobre os diversos saltos entre os artigos de origem. Considerando que há uma vasta informação na Wikipédia sobre Taylor Swift, também recebemos um escore de Cobertura da KB de 99%.

    Expandindo sua Base de Conhecimento

    Em última análise, você pode conectar virtualmente qualquer fonte de conhecimento à sua instância NeuralSeek para geração de respostas via os conectores de KB Virtual no mAIstro. Você pode escolher entre uma variedade de conectores de banco de dados, conectores de Base de Conhecimento ou conectores de Pesquisa na Web. Ou, conecte-se a qualquer fonte adicional via nosso conector de API Rest.

    Construindo um Fluxo

    1. Navegue até o mAIstro em sua instância NeuralSeek.
    2. Selecione o nó Virtual KB - In no menu lateral, em RAG Tools.

    Este nó lhe fornece várias variáveis para usar dentro do seu fluxo.

    1. Selecione o nó Website Data no menu lateral, em Get Data. Isso irá se conectar automaticamente abaixo do seu primeiro nó.
    2. Clique no ícone de engrenagem para inserir qualquer URL válido. Neste exemplo, estamos nos conectando a uma pesquisa do Google: https://www.google.com/search?gfns=1&q=<< name: virtualKbIn.contextQuery>>
    3. Selecione o nó Set Variable no menu lateral, em Control Flow.
    4. Clique e arraste o nó Set Variable para a direita do nó Website Data para encadeá-lo.
    5. Clique no ícone de engrenagem para definir o nome da variável. Neste exemplo, o nome da variável é google.

    A adição da variável virtualKbIn.contextQuery permite que o contexto da consulta do usuário seja transportado dinamicamente na pesquisa do Google.

    1. Selecione um segundo nó Website Data.
    2. Clique no ícone de engrenagem para inserir qualquer URL adicional. Neste exemplo, estamos nos conectando à página de documentação do NeuralSeek: https://documentation.neuralseek.com/
    3. Selecione o nó Set Variable no menu lateral, em Control Flow.
    4. Clique e arraste o nó Set Variable para a direita do segundo nó Website Data para encadeá-lo.
    5. Clique no ícone de engrenagem para definir o nome da variável. Neste exemplo, o nome da variável é docs.

    Adicionamos a documentação do NeuralSeek como uma segunda fonte de referência para nossa Base de Conhecimento e estamos realizando uma extração estática das informações do site.

    1. Selecione o nó Virtual KB - Out no menu lateral, em RAG Tools.
    2. Clique no ícone de engrenagem para configurar as informações a serem enviadas de volta para Seek. Neste exemplo, queremos definir o trecho incluindo os nomes das variáveis: << name: google >>\n<< name: docs >>.
    3. Adicionalmente, podemos predefinir o kbCoverage, kbScore, url e o nome do documento. Neste exemplo, definimos o nome do documento como Virtual KB.
    4. Salve seu fluxo mAIstro com um nome único e uma descrição opcional. Neste exemplo, o nome é websiteKB. Ambos os sites agora serão puxados ao vivo sempre que um Seek for enviado. As informações raspadas dos sites sairão dinamicamente e em paralelo, depois serão reintegradas ao processo Seek para geração de respostas.

    Nota

    Embora usemos um único documento concatenado aqui por simplicidade, é possível dividi-lo em vários documentos. Basta criar um objeto JSON com uma matriz de objetos de documento contendo as propriedades: documento (título), url, pontuação e passagem.

    {{ virtualKbIn }} {{ web | url: https://www.google.com/search?gfns=1&q=<< name: virtualKbIn.contextQuery>> }}=>{{ variable | name: google }} {{ web | url: https://documentation.neuralseek.com/ }}=>{{ variable | name: docs }} {{ virtualKbOut | context: << name: google >>\n<< name: docs >> | kbCoverage: 0 | kbScore: 0 | url: | documento: Virtual KB }}

    Configurando um Virtual KB

    1. Navegue até a guia Configurar em sua instância do NeuralSeek.
    2. Expanda o accordion Conexão com o Banco de Conhecimento.
    3. Para o Tipo de Banco de Conhecimento, selecione a opção Virtual KB.
    4. Para o modelo Virtual KB do mAIstro, selecione a opção websiteKB.
    5. Clique no ícone vermelho Salvar na parte inferior da tela para salvar sua configuração.

    Buscar com um Virtual KB

    1. Navegue até a guia Buscar em sua instância do NeuralSeek.
    2. Digite qualquer pergunta. Por exemplo, O NeuralSeek oferece um Hands-On Lab?
    3. Clique no botão Buscar para gerar uma resposta.

    Podemos expandir a fonte do Virtual KB abaixo do Contexto do Banco de Conhecimento e ver quais informações foram extraídas da Pesquisa do Google e quais foram extraídas da URL da Documentação do NeuralSeek para gerar a resposta.
