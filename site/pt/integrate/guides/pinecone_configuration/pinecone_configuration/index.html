    Integração do Pinecone com o NeuralSeek

    Este guia fornece instruções passo a passo para configurar o Pinecone como a base de conhecimento e usá-lo junto com o modelo de incorporação. Além disso, é fornecida uma explicação técnica de como essa configuração funciona. Um exemplo de script Node.js para carregar documentos no índice do Pinecone também está incluído.

    Embora este guia se concentre no Pinecone, vale a pena notar que você também pode usar o Milvus como um banco de dados de vetores alternativo.

    Pré-requisitos

    • Certifique-se de ter o Node.js instalado.

    Etapas

    1. Criar uma conta Pinecone

    • Acesse o Pinecone e crie uma nova conta.

    2. Criar um novo índice no Pinecone

    • Navegue até o painel e crie um novo índice.

    • Dependendo do modelo de incorporação que você planeja usar, escolha o tamanho de vetor apropriado:

      • text-embedding-ada-002: Tamanho do vetor 1536
      • text-embedding-3-small: Tamanho do vetor 1536
      • text-embedding-3-large: Tamanho do vetor 3072
      • infloat-e5-small-v2: Tamanho do vetor 384

    3. Configurar o NeuralSeek

    3.1. Configurar a conexão da base de conhecimento

    • Acesse a plataforma NeuralSeek.
    • Vá para a guia Configurar e configure a conexão da base de conhecimento:
    • Tipo de base de conhecimento: Pinecone
    • Idioma da base de conhecimento: Inglês
    • Nome do índice do Pinecone: docs
    • Namespace do índice do Pinecone: ns1
    • Chave da API do Pinecone: sua-chave-de-api-do-pinecone
    • Campo de dados de curadoria: text
    • Campo de nome do documento: title
    • Campo de filtro: title
    • Campo de link: link
    • Recursos de atributo: habilitado

    3.2. Adicionar um modelo de incorporação

    • Vá para a seção Modelos de incorporação e adicione uma nova incorporação:

    • Escolha a plataforma (seja Azure, NeuralSeek ou OpenAI).

    • Selecione o modelo de incorporação apropriado:
      • Para OpenAI e Azure:
      • text-embedding-ada-002: Tamanho do vetor 1536
      • text-embedding-3-small: Tamanho do vetor 1536
      • text-embedding-3-large: Tamanho do vetor 3072
      • Para NeuralSeek:
      • infloat-e5-small-v2

    4. Adicionar documentos ao índice do Pinecone via script Node.js

    4.1. Instalar pacotes necessários

    npm install axios fs path @pinecone-database/pinecone @langchain/openai

    import axios from axios;
import fs from fs;
import path from path;
import { Pinecone } from @pinecone-database/pinecone;
import { OpenAIEmbeddings } from @langchain/openai;

const folder = ./docs;

const pc = new Pinecone({
  apiKey: sua-chave-de-api-do-pinecone, // Substitua pela sua chave de API do Pinecone
});

var kb = {};
var ids = [];

const openaiAPIKey = sua-chave-de-api-do-openai; // Substitua pela sua chave de API do OpenAI

kb.importFiles = async (model, pineconeIndex, pineconeNamespace) => {
  var pineconeData = [];

  let fileList = fs.readdirSync(folder);
  var vectors = null;
  for (const file of fileList) {
    const data = JSON.parse(fs.readFileSync(path.join(folder, file)));

    if (model == infloat-e5-small-v2) {
      const embeddings = await axios.post(http://url.com, {
        text: data.text,
      });
      vectors = embeddings.data;
    } else if (
      model == text-embedding-ada-002 ||
      model == text-embedding-3-small ||
      model == text-embedding-3-large
    ) {
      var embedV2 = new OpenAIEmbeddings({
        openAIApiKey: openaiAPIKey,
        modelName: model,
      });

      vectors = await embedV2.embedQuery(data.text);
    } else {
      throw new Error(`Modelo não suportado ${model}`);
    }

    const id = data.title;
const metadata = {
      text: data.text,
      title: data.title,
      link: data.source_link,
    };
    const values = vectors;
    var record = { id, values, metadata };
    pineconeData.push(record);
    ids.push(id);
  }
  const index = pc.index(pineconeIndex);

  await index.namespace(pineconeNamespace).upsert(pineconeData);
};

    Incorporações de texto

    • Converte dados de texto em representações vetoriais densas que capturam informações semânticas.

    Correspondência de Similaridade

    • Compara vetores de consulta com vetores de documentos para encontrar as respostas mais relevantes.

    Compreensão Contextual

    • Aproveita múltiplas camadas para entender e gerar respostas contextualmente precisas.

    Fluxo de Trabalho de Integração

    1. Ingestão de Dados: Os documentos são ingeridos e processados para gerar incorporações vetoriais usando o modelo de incorporação da NeuralSeek.
    2. Indexação: As incorporações vetoriais geradas são armazenadas no Pinecone, onde são indexadas para busca e recuperação eficientes.
    3. Processamento de Consulta: Quando uma consulta é inserida, a NeuralSeek converte o texto da consulta em um vetor usando o modelo de incorporação.
    4. Pesquisa e Recuperação: O vetor de consulta é comparado com os vetores de documentos no Pinecone para encontrar as correspondências mais relevantes.
    5. Geração de Resposta: Os documentos mais relevantes são recuperados do Pinecone, e a NeuralSeek formula uma resposta com base nos dados recuperados.

    Benefícios desta Configuração

    Eficiência

    • Combinando os recursos eficientes de pesquisa vetorial do Pinecone com as poderosas incorporações da NeuralSeek, garante-se respostas rápidas e precisas.

    Escalabilidade

    • O Pinecone pode escalar para lidar com grandes volumes de dados, enquanto as incorporações da NeuralSeek mantêm alto desempenho.

    Precisão

    • As incorporações contextuais da NeuralSeek melhoram a precisão das respostas, fornecendo informações relevantes e precisas.

    Solução de Problemas

    Problema: O modelo não está fornecendo respostas precisas

    • Solução: Verifique os parâmetros do modelo e certifique-se de que o conteúdo da base de conhecimento esteja atualizado.

    Problema: Erros de upload

    • Solução: Certifique-se de que os formatos de arquivo estejam corretos e a integridade dos dados seja mantida.

    Problema: Problemas de integração

    • Solução: Verifique novamente a ligação entre o modelo e a base de conhecimento, e certifique-se de que a sincronização esteja configurada corretamente.
