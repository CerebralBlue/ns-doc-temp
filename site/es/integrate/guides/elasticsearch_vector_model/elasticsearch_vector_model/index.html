    Configurando ElasticSearch para la búsqueda vectorizada

    Descripción general

    Esta guía proporciona instrucciones paso a paso sobre cómo configurar la búsqueda vectorial con ElasticSearch. Incluye iniciar sesión en los entornos, crear claves para el acceso a la API, configurar una instancia de aprendizaje automático, descargar los modelos necesarios, crear índices de origen y destino, e ingerir datos para generar incrustaciones de texto. La guía también cubre los pasos de carga de datos manual y el uso de funciones auxiliares del cliente para la ingesta de datos. Concluye con la verificación de los datos y las incrustaciones de contenido en el índice de destino.

    Iniciar sesión en los entornos

    Comience iniciando sesión en su cuenta de IBM Cloud

    • Para aprovisionar en IBM Cloud:,
      • Navegue a Bases de datos para ElasticSearch.
      • Seleccione la Edición de base de datos de platino.
    • De lo contrario, aprovisione dentro de Elastic Cloud como de costumbre.

    Hay dos entornos con los que trabajar.

    • Consola de ElasticSearch Cloud. Observe los iconos en la esquina superior derecha.
    • Consola de Kibana
      • Es posible que los usuarios sean llevados directamente a la consola de Kibana después de crear un despliegue. De lo contrario, navegue allí seleccionando Abrir en la página de despliegue desde la consola de ElasticSearch Cloud.

    Creación de claves

    • Seleccione el icono del círculo en la esquina superior derecha de la pantalla de Kibana.
    • Seleccione Detalles de conexión
    • Aquí verá el punto final de ElasticSearch y el ID de nube.
    • Seleccione Crear y administrar claves de API.
    • Para crear una nueva clave de API, haga clic en Crear clave de API.
      • Agregue un nombre único.
      • Seleccione el tipo como Clave de API de usuario.
      • Haga clic en el botón Crear clave de API en la parte inferior del cuadro de diálogo.

    Guarde estos valores en un lugar seguro para su uso posterior.

    Crear una instancia de aprendizaje automático

    Elastic requiere una instancia de aprendizaje automático para ejecutar los modelos de NLP necesarios para vectorizar los datos para su indexación.

    • Navegue a la pantalla de inicio de su instancia de ElasticSearch.
    • Navegue al despliegue recién creado y seleccione Administrar.
    • En el menú lateral, seleccione Editar.
    • Desplácese hacia abajo hasta la sección Instancias de aprendizaje automático.
    • Seleccione Agregar capacidad.
    • Seleccione 4 GB de RAM.
    • Haga clic en Guardar en la parte inferior de la página.

    Descargar modelos

    Abstract
    • En Kibana, haga clic en el icono del menú en la esquina superior izquierda y navegue a Analítica > Aprendizaje automático > Modelos entrenados.
    • Haga clic en el botón Descargar en la columna Acciones
      • Elija el modelo recomendado .elser_model_2_linux-x86_64
      • Puede tardar un tiempo en finalizar la descarga.
    • Haga clic en el enlace Implementar que aparece cuando se pasa el mouse sobre el modelo descargado.
    • Deje los ajustes predeterminados en la columna Diálogo y seleccione Iniciar.
    • La columna Estado mostrará Implementado cuando se haya realizado correctamente.

    Abstract

    Se recomienda usar Eland para cargar y descargar el modelo deseado en ElasticSearch.

    • Ejecute este comando para instalar el cliente de Python Eland con PyTorch: python -m pip install 'eland[pytorch]'
    • Ejecute este script para descargar el modelo de Hugging Face, convertirlo al formato TorchScript y cargarlo en el clúster de Elasticsearch:

    ``` eland_import_hub_model

    --cloud-id \ -u -p \ --hub-model-id elastic distilbert-base-cased-finetuned-conll03-english \ --task-type ner ```

    - Especifique el **identificador de Elastic Cloud** utilizando la configuración de TLS con un certificado descargado de **IBM Cloud -> Database for Elasticsearch -> pestaña Información general**.
- Proporcione los detalles de autenticación para acceder a su clúster.
- Especifique el **identificador** del modelo en el Hugging Face model hub.
- Especifique el **tipo de tarea NLP** como `text_embedding`.

> Se recomienda utilizar el modelo `intflost/multilingual-e5-base` de Hugging Face para comenzar.

> Puede llevar tiempo que el modelo se inicie automáticamente, hasta unas horas.

    Crear el índice de origen y cargar los datos

    Los índices se pueden crear cargando manualmente los datos utilizando la API _bulk, o utilizando una función auxiliar del cliente que creará el índice y cargará los datos.

    Abstract
    • Navegue a la consola de Kibana.
    • En el menú lateral, seleccione Administración > Herramientas de desarrollo para abrir la consola de desarrollo.
    • Elimine cualquier código que aparezca.
    • Para crear el índice de origen, ingrese el siguiente código:
        PUT /search-gs-docs-src
    {
    mappings: {
        properties: {
        title: { 
            type: text 
        },
        content: { 
            type: text 
        },
        source: { 
            type: text 
        },
        url: { 
            type: text 
        },
        public_record: { 
            type: boolean 
        }
        }
    }
    }
    • Haga clic en el icono ejecutar.
    • Prepare los datos para la ingesta a granel convirtiendo manualmente los datos y utilice la consola de desarrollo para cargarlos ingresando el siguiente código:
        POST _bulk
    { index : { _index : search-gs-docs-src, _id : 1 } }
    { title : Top 3 Best Practices to Secure Your Gainsight PX Subscription,
    content : We should all protect what has been entrusted…",
    url : https://support.gainsight.com/...,
    source : docs",
    "public_record":true,
    "objectID": "https://support.gainsight.com/..."
    }
    { index : { _index : search-gs-docs-src, _id : 2 } }
    { title : Using PX with Content Security Policy,
    content : This article describes the steps to allow a Content Security Policy…",
    url : https://support.gainsight.com/...,
    source : docs",
    "public_record":true,
    "objectID": "https://support.gainsight.com/..."
    }
    …
    Abstract
    • Ingrese el siguiente código para utilizar la función auxiliar del cliente para crear el índice y cargar los datos:

    ``` 'use strict'

    require('array.prototype.flatmap').shim()
const { Client } = require('@elastic/elasticsearch')
const client = new Client({
cloud: { id: '<cloud_id>'},
auth: { apiKey: '<api_key>' }
})
const dataset = require('./gainsight_documentation_data/gainsight-en-federated.json')

// Crear y cargar el índice de origen
async function run () {
await client.indices.create({
    index: 'search-gs-docs-src',
    operations: {
    mappings: {
        properties: {
        title: { type: 'text' },
        content: { type: 'text' },
        url: { type: 'text' },
        source: { type: 'text' },

    registro_público: { type: 'boolean' }, objectID: { type: 'text' } } } } }, { ignore: [400] })

        const operaciones = dataset.flatMap(doc => [{ index: { _index: 'search-gs-docs-src' } }, doc])

    const respuestaGranel = await client.bulk({ refresh: true, operaciones })

    if (respuestaGranel.errors) {
        const documentosConErrores = []
        // La matriz de elementos tiene el mismo orden del conjunto de datos que acabamos de indexar.
        // La presencia de la clave `error` indica que la operación
        // que hicimos para el documento ha fallado.
        respuestaGranel.items.forEach((acción, i) => {
        const operación = Object.keys(acción)[0]
        if (acción[operación].error) {
            documentosConErrores.push({
            // Si el estado es 429 significa que puede volver a intentar el documento,
            // de lo contrario es muy probable que sea un error de asignación, y debe
            // corregir el documento antes de volver a intentarlo.
            estado: acción[operación].status,
            error: acción[operación].error,
            operación: operaciones[i * 2],
            documento: operaciones[i * 2 + 1]
            })
        }
        })
        console.log(documentosConErrores)
    }

    const count = await client.count({ index: 'search-gs-docs-src' })
    console.log(count)
    }

    run().catch(console.log)
    • Para obtener el nombre de la canalización con el modelo cargado, navegue a Kibana > Machine Learning > Modelos entrenados.
    • Expanda el modelo implementado.
    • Navegue a la pestaña Canalizaciones para ver la my-content-embesddings-pipeline creada en el paso anterior.

    Para confirmar que se ejecutó la tarea con éxito, ejecute el siguiente comando utilizando el ID de tarea producido en la respuesta del comando anterior. GET _tasks/<task_id>.

    • Verifique que los incrustaciones de contenido se encuentren en el nuevo índice de destino.
      • Navegue a Kibana.
      • Navegue a Búsqueda > Contenido > Índices.
      • Abra el índice search-gs-docs-dest.
      • Abra la pestaña Documentos para ver los datos.

    Asignar un campo

    Los modelos compatibles con ElasticSearch NLP generan vectores densos como salida, por lo que el tipo de campo dense_vector para el índice es adecuado para almacenarlos. Este tipo de campo debe configurarse con el mismo número de dimensiones utilizando la opción dims.

    • Ingrese el siguiente código en la consola de desarrollo para crear una asignación de índice que defina el campo que contiene la salida del modelo. 

          PUT my-index
    {
    mappings: {
        properties: {
        my_embeddings.predicted_value: { 
            type: dense_vector, 
            dims: 384 
        },
        my_text_field: { 
            type: text 
        }
        }
    }
    }

    • my_embeddings.predicted_value es igual al nombre del campo que contiene los incrustaciones generados por el modelo.

    • El campo type debe ser dense_vector.
    • El campo dims contiene el número de dimensiones de los incrustaciones producidos por el modelo. Asegúrese de que este número esté configurado en el campo dense_vector.
    • El campo my_text_field es igual al nombre del campo a partir del cual crear la representación de vector denso.
    • El campo type es text.

    Probar la búsqueda semántica

    Abstract

    Pruebe la búsqueda semántica utilizando la consulta text_expansion proporcionando el texto de la consulta y el ID del modelo ELSER.

    • Ingrese el siguiente código en la consola de desarrollo:
            GET search-gs-docs-dest/_search
    {
    query:{
        text_expansion:{
            content_embedding:{
                model_id:.elser_model_2_linux-x86_64,
                model_text:Ingrese la consulta de muestra aquí
            }
        }
    }
    }
    • El campo content_embedding contiene la salida ELSER generada.
    Abstract

    Los modelos de vector denso permiten a los usuarios consultar características de clasificación con una búsqueda kNN. En el campo knn, los usuarios proporcionarán el nombre del campo de vector denso. En el campo query_vector_builder, agregue el ID del modelo y el texto de la consulta.

    • Ingrese el siguiente código en la consola de desarrollo:
        GET my-index/_search
    {
    knn: {
        field: my_embeddings.predicted_value,
        k: 10,
        num_candidates: 100,
        query_vector_builder: {
        text_embedding: {
            model_id: sentence-transformers__msmarco-minilm-l-12-v3,
            model_text: la cadena de consulta
        }
        }
    }
    }

    Conectar NeuralSeek a Elasticsearch

    • Navegue a su cuenta de IBM Cloud.
    • Abra la instancia de servicio de NeuralSeek.
    • Navegue a la pantalla Configurar.
    • Guarde su configuración actual haciendo clic en el botón Descargar configuración en la parte inferior de la pantalla.
    • Abra el acordeón Conexión de la base de conocimiento y actualice los siguientes campos.
      • Establezca el tipo de base de conocimiento en ElasticSeach
      • Establezca el punto final de ElasticSearch.
      • Establezca la clave de API privada de ElasticSearch.
    • Establezca el Nombre del índice de ElasticSearch en el índice de destino. En este caso, search-gs-docs-dest.
      • Establezca el Campo de datos de curaduría en content.
      • Establezca el Campo de nombre de documentación en title.
      • Establezca el Campo de enlace en url.
    • Haga clic en el botón Guardar en la parte inferior de la página.

    Habilitar la búsqueda vectorial en NeuralSeek

    En la pantalla de configuración de NeuralSeek, abra el acordeón de Configuración de búsqueda híbrida y vectorial para actualizar los siguientes campos.

    • Establezca el tipo de consulta de Elastic en Híbrido.
      • Esto permitirá tanto la búsqueda Lucene (coincidencia exacta) como la Vectorial (semántica) para lograr una respuesta más sólida.
    • Establezca el ID del modelo en .elser_model_2_linux-x86_64
    • Establezca el Campo de incrustación en content_embedding
    • Establezca el campo Usar el modelo ELSER de Elastic en True para usar el modelo ELSER, o establézcalo en False para permitir que NeuralSeek espere un formato JSON para una consulta de búsqueda kNN.
    • Haga clic en Guardar en la parte inferior de la pantalla.

    !!! warning Si se usa 'IBM Databases for ElasticSearch'

    Con la búsqueda híbrida, KnnScoreDocQuery se creó mediante un lector diferente. Para solucionar esto, ingrese el siguiente código en la consola de desarrollo de Kibana:
```
    PUT /<INDEX_NAME>/_settings
    {
        index : {
            highlight.weight_matches_mode.enabled : false
        }
    }
```
