    Detalhes da configuração

    Visão geral

    Este local do NeuralSeek é onde os usuários podem editar as configurações dos recursos do NeuralSeek. Existem dois tipos de configurações: Configurações Padrão e Configurações Avançadas.

    Configurações Padrão

    Essas opções estão disponíveis por padrão, após a provisão do NeuralSeek em uma nova instância. Você pode usar o botão Mostrar Opções Avançadas na tela Configurar para mostrar mais / configurações avançadas.

    Conexão com o Banco de Conhecimento

    Os usuários podem alterar o tipo de Banco de Conhecimento, juntamente com a URL associada, chaves de API, ID do projeto e outras informações relevantes. Use as setas suspensas para configurar manualmente os campos do esquema no Banco de Conhecimento conectado.

    • Tipo de Banco de Conhecimento: O provedor do Banco de Conhecimento.
    • Idioma do Banco de Conhecimento: O idioma dos documentos carregados no Banco de Conhecimento selecionado.
    • Notas: Opcionalmente, adicione quaisquer notas relacionadas à configuração do Banco de Conhecimento selecionado.
    • Campo de Dados de Curadoria: Selecione o parâmetro do conteúdo/corpo do documento da sua FAQ.
    • Campo de Link: Selecione o campo de URL dos metadados do documento - mostrado abaixo do título ou servido na bolha de bate-papo do Agente Virtual como um link.
    • Campo Nome do Documento: O campo de metadados do documento para o Título do documento.
    • Atribuir fontes dentro do Contexto LLM por Nome do Documento: Os usuários têm a opção de habilitar ou desabilitar a atribuição de fontes dentro do contexto LLM pelo nome do documento. Por exemplo, quando desabilitado, a saída será formatada apenas com o conteúdo do documento. Quando habilitado, a saída será formatada com uma frase introdutória que atribui o 'conteúdo do documento' ao 'nome' do documento apropriado (por exemplo, O documento 'nome' afirma que: 'conteúdo do documento'). Isso ajuda alguns LLMs a acompanhar o rastro das informações.
    • Campo de Filtro: Selecione o campo de metadados do documento a ser usado para filtragem. Por exemplo, você pode filtrar um campo 'tipo_documento' apenas para tipos 'PDF'.
    • Reordenar Lista de Valores: Os usuários podem inserir uma lista priorizada de valores que desejam reordenar acima de outros resultados, independentemente da pontuação do KB. Clique no ícone da lâmpada para adicionar uma nova linha e inserir o valor da prioridade.

    Detalhes do LLM

    Aqui os usuários podem adicionar um LLM de sua escolha e definir ou modificar as configurações do modelo LLM. Ao clicar em Adicionar um LLM, os usuários serão solicitados a selecionar um LLM de sua plataforma de escolha e inserir as informações relevantes, como chaves de API, URLs de ponto de extremidade, IDs de projeto, etc. Use a seta suspensa para habilitar os idiomas de sua escolha: existem um total de 56 idiomas LLM suportados. Os usuários também podem modificar quais funções do NeuralSeek habilitar para o LLM adicionado. Observe que você deve adicionar pelo menos um LLM. Se você adicionar vários, o NeuralSeek fará o balanceamento de carga entre eles para as funções selecionadas que têm vários LLMs. Os recursos que um LLM não é capaz de não estarão disponíveis para seleção. Se você não fornecer um LLM para uma função, não haverá fallback e essa função do NeuralSeek será desativada.

    • Chave de API/Acesso: A chave de API ou de acesso do provedor de LLM / Serviço.
    • Chave Secreta/Zen: A chave secundária do provedor de serviços (apenas para alguns provedores)
    • Ponto de Extremidade: A URL do ponto de extremidade para o serviço selecionado.
    • Região: A região onde o serviço LLM está provisionado.
    • ID do Projeto: O ID do projeto para o espaço de trabalho LLM.
    • Idiomas LLM: Habilite ou desabilite idiomas específicos a serem usados com o LLM selecionado.
    • Funções do LLM: Ativar ou desativar funções específicas para cada LLM, essencialmente selecionando qual LLM usar para cada tarefa ou permitindo o balanceamento de carga para tarefas específicas. Por exemplo, uma opção é usar um LLM específico para seek e um LLM diferente para maistro ou translate, permitindo casos de uso flexíveis e específicos.
    • ID do LLM: O ID/nome interno do LLM selecionado. Use esse ID em chamadas de API ou do mAIstro.
    • Teste: Execute um teste de conclusão no LLM, verificando as credenciais. Isso não 'salva', você ainda deve 'salvar' suas configurações com o botão da interface principal.
    • Excluir: Remova o LLM selecionado da configuração.

    Nota

    Esta seção está disponível apenas se você estiver usando o plano BYOLLM (traga seu próprio LLM) do NeuralSeek.

    Nem todos os provedores de LLM são iguais - Todas as opções são listadas aqui, mesmo que seu provedor não precise desses parâmetros específicos.

    Preferências da Empresa/Organização

    Aqui você pode configurar o nome da sua empresa e a descrição do que a empresa se concentra principalmente.

    • Nome da Empresa ou Organização: Este campo é usado para ajudar a alinhar as consultas do usuário ao KB da empresa. Por exemplo, "Como eu uso seu produto?" irá direcionar para este valor.
    • Afinidade de Resposta da Empresa: Ativar para adicionar afinidade à sua empresa ou marca, além do texto já existente no seu Conhecimento Base e Discursos Preparados.
    • Discurso Preparado: Efetivamente um documento sempre fixado que é incluído na documentação para cada chamada seek. Isso ajuda a responder perguntas como uma fonte de conhecimento de reserva quando a pesquisa do usuário não produz documentação relevante.

    Preferências da Plataforma

    Seu balcão único para todas as preferências relacionadas à plataforma. Defina tempos limite, configure links incorporados, formato de saída do Agente Virtual, etc.

    • Tempo Limite: Defina isso alguns segundos a menos que o tempo limite da sua plataforma de chatbot. Quando o tempo limite for atingido, o NeuralSeek tentará responder com uma resposta armazenada em cache, se disponível.
    • Turnos de Contexto: O número de turnos na conversa a serem passados para o LLM. Aumentar isso não é recomendado, pois reduzirá o contexto do LLM disponível para sua documentação.
    • Idioma de Saída Padrão: O idioma para responder. Definir o valor do idioma na API substituirá este parâmetro. Defina como Corresponder entrada para tentar identificar o idioma de entrada e responder nesse idioma detectado.
    • Traduzir para o Idioma do KB: Quando ativado, traduz consultas para o idioma selecionado no Conhecimento Base.
    • Tipo de Agente Virtual: Selecione o tipo de Agente Virtual para curadoria de respostas. Este é o formato que o NeuralSeek usará para criar o arquivo do chatbot.
    • Incorporar links nas respostas retornadas: Ativar para incorporar links clicáveis nas respostas geradas pelo seek no lado da API.
    • Lista Personalizada de Stopwords: Stop Words - Uma lista de palavras não úteis ou insignificantes a serem removidas no pré-processamento. Adicione palavras aqui para substituir a lista de stopwords do NeuralSeek.

    Categorização de Intenção

    Crie tipos de categorias e com descrições em linguagem natural para controlar como as intenções em perguntas de usuários podem ser categorizadas. Geralmente, uma pergunta seria automaticamente categorizada como FAQ, mas você pode fornecer categorias personalizadas adicionais aqui.

    • Categoria: O nome da categoria.
    • URL/Link: O link a ser retornado para respostas nesta categoria. Isso substitui a URL retornada da fonte de documentação.
    • Descrição: Uma descrição em linguagem natural da categoria. Por exemplo, raça de cachorro, como Yorkshir Terrier ou Labrador.
    • Ação de Palavra Bloqueada: Remova as palavras ofensivas da entrada do usuário ou bloqueie a pergunta completamente.
    • Lista de Palavras Bloqueadas: Digite palavras ou frases (separadas por vírgulas) que não são permitidas na entrada do usuário. Isso é útil para bloquear nomes específicos de clientes ou produtos concorrentes, bem como outras palavras sensíveis não abrangidas pelo corpus base do NeuralSeek.

    Informações de Identificação Pessoal (PII)

    Os usuários podem definir como lidar com quaisquer dados de PII detectados que tenham sido incluídos na pergunta.

    • Ação a ser tomada: Especifique as ações quando o PII for detectado:
      • Mascarar: Mascarar e armazenar o PII quando for detectado na entrada do usuário. A máscara ocultará o PII em todos os locais.
      • Sinalizar: Sinalizar e armazenar o PII quando for detectado na entrada do usuário. O PII será sinalizado em todos os locais.
      • Nenhuma Ação: Nenhuma ação será tomada quando o PII for detectado na entrada do usuário. Ele será armazenado em texto simples.
      • Ocultar (reter para Análise): Ocultar (mascarar) o PII quando for detectado na entrada do usuário, mas manter o PII no banco de dados para Análise.
      • Excluir (incluindo da Análise): Excluir o PII completamente quando for detectado na entrada do usuário, incluindo da Análise armazenada.
    • Confiar em palavras encontradas em documentos de origem: Indicar se certos termos confiáveis em documentos de origem devem ser reconhecidos ou ignorados.
    • Filtros de PII Pré-LLM: Esses são executados dinamicamente na entrada do usuário antes de ser enviada para o LLM ou KnowledgeBase. Clique no ícone da lâmpada para adicionar uma descrição, como um número de telefone e uma expressão regular correspondente.
    • Filtros de PII Baseados em LLM: Esses usam o LLM escolhido para identificar o PII. Clique no ícone da lâmpada para adicionar uma frase de exemplo e os elementos de PII correspondentes, separados por vírgulas.
    • Detectores de PII do NeuralSeek: Selecione os detectores padrão do NeuralSeek para capturar o PII.

    Filtro de Palavrões / Filtro de HAP (Ódio, Abuso, Palavrões)

    Os usuários podem ativar ou desativar o filtro de palavrões, bem como adicionar um texto para responder a perguntas sensíveis que são bloqueadas.

    • Ativar filtro de palavrões: Escolha qual filtro usar para filtragem de palavrões. Você pode usar o endpoint de moderação do LLM, se disponível, o Filtro NeuralSeek ou desativá-lo.
    • Texto de resposta bloqueado: O texto a ser exibido quando a entrada ou a pergunta for bloqueada. Por exemplo, Essa parece ser uma pergunta sensível.

    Proteção de Atribuição

    • Ajuste a tolerância a desinformação para gerar texto sobre a empresa ou associar pessoas ou coisas que não tenham referências específicas no material da KnowledgeBase usando a escala deslizante de Rígido a Padrão. Quanto mais rígidas as suas configurações, maior a chance de bloquear ocasionalmente perguntas legítimas que usam palavras alternativas ou são mal documentadas em sua KnowledgeBase.

    Confiança de Aviso

    Use a escala deslizante para aumentar o limite de confiança.

    • Confiança %: Quaisquer respostas abaixo deste número terão o texto abaixo pré-anexado à resposta dada.
    • Texto de aviso: O texto a ser pré-anexado à resposta dada.

    Confiança Mínima

    Use a escala deslizante para aumentar o percentual de confiança mínima e o percentual de confiança mínima para exibir uma URL. Adicione um texto para responder com perguntas que não atendam à confiança mínima e selecione se deseja adicionar um fallback de URL no mínimo. (por exemplo, não há nada em nossa base de conhecimento sobre isso.).

    • Percentual de confiança mínima: Quaisquer respostas inferiores a este número terão o texto abaixo substituído no lugar da resposta.
    • Modelo mAIstro Opcional - Um modelo mAIstro para lidar com a confiança mínima de uma maneira personalizada.
    • Texto de resposta: A resposta a ser dada quando as respostas estiverem abaixo do percentual de confiança mínima definido.
    • Percentual de confiança mínima para exibir uma URL: Quaisquer respostas inferiores a este número não retornarão uma URL vinculada.
    • Fallback de URL Opcional - Uma URL a ser oferecida quando a confiança mínima não for atendida.

    Configuração Avançada

    Estas opções (expandidas) estão disponíveis após ativar Mostrar opções avançadas da configuração padrão.

    Ajuste da Base de Conhecimento

    Ajustar sua Base de Conhecimento é uma parte importante de criar um sistema com bom desempenho.

    • Intervalo de Pontuação de Documentos: O intervalo de pontuação superior dos documentos a serem retornados. Por exemplo, quando definido como 0,8 ou 80%, retorne os 80% principais documentos com maior pontuação, descartando os 20% com menor pontuação. Quanto menor o número, mais rígido será o limite de pontuação. Geralmente é melhor definir um número alto, usado com a configuração de Máx. Docs.
    • Penalidade de Data do Documento: Penalizar as pontuações de documentos que incluem datas antigas. Um número maior significa uma penalidade maior para documentos mais antigos, escalando com o tempo/idade.
    • Cache de Consulta da KB: Limitar consultas repetidas à Base de Conhecimento armazenando-as em cache. Defina isso para o número de minutos que você deseja preservar as consultas da KB armazenadas em cache.
    • Máx. Documentos por Busca: O número de documentos a serem enviados para o LLM em cada ação de Busca. Geralmente os melhores resultados são vistos com isso definido entre 4-5 documentos.
    • Tamanho do Snippet: A contagem de caracteres a serem passados para a KB para o tamanho do trecho do documento. Quanto maior o número, maior o fragmento da documentação. Geralmente é melhor como um número menor - em torno de 500.
    • Pontuação Máxima Bruta: A maior pontuação de documento de todos os tempos que o NeuralSeek já viu da KB. O NeuralSeek usa esse número internamente para calcular uma pontuação de 100% para os documentos.

    Configurações de Pesquisa Híbrida e Vetorial

    Oferecendo a capacidade de escolher entre pesquisar com Lucene, Vetorial ou uma abordagem Híbrida

    • Tipo de Consulta: O tipo de consulta que o NeuralSeek usará para reunir a documentação de origem.
    • Lucene: Consultas de correspondência exata.
    • Vetorial: Pesquisa de similaridade vetorial, com base no modelo Vetorial implantado.
    • Híbrido: Uma consulta de pesquisa combinada que aumenta ligeiramente os resultados do Lucene, permitindo uma alternativa suave para os resultados Vetoriais se nenhuma correspondência exata for encontrada.
    • Usar o modelo ELSER da Elastic? O formato da consulta é diferente usando ELSER vs modelos KNN implantados. Selecione Falso se não estiver usando o modelo ELSER da Elastic.
    • ELSER - ID do Modelo: O nome do modelo implantado e em execução.
    • ELSER - Campo de Incorporação: O nome do campo de metadados onde os embeddings Vetoriais gerados são armazenados.
    • Consulta KNN Elástica: O JSON da consulta Vetorial KNN a ser executada. Existem alguns valores a serem definidos dentro do JSON:
    • field: O nome do campo de metadados onde os embeddings Vetoriais são armazenados.
    • model_id: O nome do modelo implantado e em execução.
    • model_text: Oferecemos uma variável de expansão << query >> para inserir a consulta gerada pelo NeuralSeek. Útil para editar se alguns modelos Vetoriais exigirem um formato específico, por exemplo, question: <<query>>
    • Consulte a documentação da Elastic para obter mais informações sobre os outros parâmetros disponíveis.

    Engenharia de Prompt

    Isso permite que usuários especialistas injetem instruções específicas no prompt base do LLM. A maioria dos casos de uso não precisará disso e não deve usar isso. (por exemplo, não forneça informações factuais ou atue como um agente de suporte ao cliente. O extenso prompt do NeuralSeek já faz isso.)

    Não habilite casualmente, pois você pode enfraquecer as salvaguardas e o extenso prompt que o NeuralSeek fornece pronto para uso. Não use nenhum idioma além do inglês na engenharia de prompt.

    • Instruções do Prompt: Texto a ser adicionado ao final do prompt do LLM. Isso raramente pode ser útil, mas alguns exemplos podem ser Responder com uma lista com marcadores se possível, ou Responder em dialeto de cowboy.

    Engenharia de Respostas e Preferências

    Personalizar a engenharia de respostas e definir preferências fornece adaptabilidade a diferentes contextos.

    • Verbosidade da Resposta Utilize as escalas deslizantes para definir se a geração de respostas deve se ater a ser concisa e curta, ou oferecer mais liberdade para ser flexível e verbosa.
    • Forçar Respostas da Base de Conhecimento: Ative para adicionar prompts extras para ajudar a forçar as respostas da documentação retornada. Geralmente é melhor manter isso ativado.
    • Expressões Regulares: Clique no ícone da lâmpada para adicionar uma nova linha. Digite uma expressão regular e uma substituição correspondente. Por exemplo, use esse recurso para remover ou trocar números de telefone, e-mails, etc.

    Correspondência de Intenção e Configuração de Cache

    O NeuralSeek gera e agrupa automaticamente a entrada do usuário em intenções. Quando a entrada do usuário não corresponde a uma intenção existente, a pergunta é adicionada ao grupo FAQ genérico.

    Os seguintes tipos de correspondência de intenção estão disponíveis:

    • Correspondência Exata: A entrada do usuário corresponde exatamente a uma intenção.
    • Similaridade de Vetor: Compare a similaridade de vetor da entrada do usuário com as intenções existentes, correspondendo com intenções semelhantes.
    • Utilize o recurso "Experimente" para testar a similaridade de intenção. Digite um exemplo de frase e clique no botão "Testar". A saída mostrará a intenção semelhante extraída da guia "Curar" e um escore de similaridade correspondente.
    • Utilize a escala deslizante "Limite de Correspondência de Intenção" para definir o percentual mínimo de correspondência para corresponder a uma Intenção existente.
    • Correspondência Aproximada: A entrada do usuário corresponde de perto a uma intenção, mas não exatamente.
    • Correspondência de Palavras-Chave: A entrada do usuário contém palavras-chave que correspondem exatamente às palavras-chave em uma intenção.
    • Correspondência Aproximada de Palavras-Chave: A entrada do usuário contém palavras-chave que correspondem de perto a uma intenção.

    Os usuários também podem configurar como o cache de respostas deve ser feito para respostas editadas e respostas normais. Isso é útil para acelerar os tempos de resposta e produzir resultados mais consistentes.

    • Cache de Respostas Editadas: Defina a quantidade mínima de respostas editadas antes que a geração de linguagem pare e as respostas armazenadas em cache sejam servidas. Defina a escala como '0' para desativar o cache de respostas editadas.
    • Cache de Respostas Normais: Defina a quantidade mínima de respostas geradas normalmente para armazenar antes que a geração de linguagem pare e as respostas armazenadas em cache sejam servidas. Defina a escala como '0' para desativar o cache de respostas editadas.

    Nota

    As respostas editadas têm prioridade no Cache de Respostas Normais, seguidas pela resposta gerada mais recente.

    Entendimento de Tabelas

    Isso pré-processa seus documentos para extrair e analisar dados tabulares em um formato adequado para consulta conversacional. Como esse processo de preparação é tanto oneroso quanto demorado, esse recurso é opt-in e consumirá 1 consulta seek para cada tabela pré-processada. As Coleções de Rastreamento da Web não são elegíveis para o entendimento de tabelas, pois o intervalo de re-rastreamento causará um uso excessivo de computação. O tempo de preparação da tabela leva vários minutos por página. Entre em contato com cloud@cerebralblue.com com detalhes da sua oportunidade e caso de uso para ser considerado para acesso.

    • IDs da Coleção de Descoberta: Clique no ícone da lâmpada para adicionar uma nova linha. Digite o ID da coleção desejada do Watson Discovery para preparação da tabela.

    Nota

    O Entendimento de Tabelas requer um LLM compatível com o Entendimento de Tabelas Habilitado. Nem todos os LLMs são capazes de Entendimento de Tabelas.

    Filtro de Documento Corporativo

    Conecte o NeuralSeek a um mecanismo de regras corporativas externo para filtrar a documentação permitida por usuário. Cada solicitação enviará os IDs da documentação encontrada para um endpoint que você definir aqui. Quaisquer IDs não retornados do filtro corporativo serão bloqueados.

    • Ativar Filtro Corporativo: Se ativado, preencha todas as informações relevantes, incluindo:
    • URL base para o filtro corporativo (get): O URL do mecanismo de filtro de documentos corporativos.
    • Parâmetro de URL para o Nome de Usuário: O nome do parâmetro para a ID do usuário.
    • Parâmetro de URL para o campo KB: O nome do parâmetro para a ID do documento para filtragem de permissões.
    • Campo da Base de Conhecimento a enviar: O campo de metadados KB a ser enviado como a ID do documento.

    Registro Corporativo

    Conecte o NeuralSeek a um endpoint de registro de auditoria corporativa. Quando conectado e ativado, todas as solicitações e respostas para o endpoint da API Seek, bem como a guia Curate, serão registradas em sua instância do Elasticsearch. Os usuários podem registrar o prompt completo do LLM Seek para fins de auditoria e conformidade.

    • Ativar Registro Corporativo: Alterne o ícone para Ativar ou Desativar este recurso. Se ativado, preencha todas as informações relevantes, incluindo: Endpoint do Elasticsearch e Chave da API do Elasticsearch.
    • Registro de Prompt: Digite "concordo" na caixa fornecida para concordar com o Contrato de Não Divulgação fornecido e ativar o registro de prompt.
