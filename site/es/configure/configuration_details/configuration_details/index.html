    Detalles de la configuración

    Información general

    Esta ubicación de NeuralSeek es donde los usuarios pueden editar las configuraciones de las funciones de NeuralSeek. Hay dos tipos de configuraciones: Configuraciones predeterminadas y Configuraciones avanzadas.

    Configuraciones predeterminadas

    Estas opciones están disponibles de forma predeterminada, al aprovisionar NeuralSeek en una nueva instancia. Puede usar el botón Mostrar opciones avanzadas en la pantalla Configurar para mostrar más / configuraciones avanzadas.

    Conexión de la base de conocimientos

    Los usuarios pueden cambiar el tipo de base de conocimientos junto con la URL asociada, las claves de API, el ID de proyecto y otra información relevante. Use las flechas desplegables para configurar manualmente los campos del esquema en la base de conocimientos conectada.

    • Tipo de base de conocimientos: El proveedor de la base de conocimientos.
    • Idioma de la base de conocimientos: El idioma de los documentos cargados en la base de conocimientos seleccionada.
    • Notas: Opcionalmente, agregue cualquier nota relacionada con la configuración de la base de conocimientos seleccionada.
    • Campo de datos de curaduría: Seleccione el parámetro del contenido/cuerpo del documento de su FAQ.
    • Campo de enlace: Seleccione el campo URL de los metadatos del documento, que se muestra debajo del título o se sirve en la burbuja de chat del Agente Virtual como un enlace.
    • Campo de nombre de documento: El campo de metadatos del documento para el título del documento.
    • Atribuir fuentes dentro del contexto de LLM por nombre de documento: Los usuarios tienen la opción de habilitar o deshabilitar la atribución de fuentes dentro del contexto de LLM por nombre de documento. Por ejemplo, cuando está deshabilitado, la salida se formateará solo con el contenido del documento. Cuando está habilitado, la salida se formateará con una oración introductoria que atribuye el 'contenido del documento' al 'nombre' de documento apropiado (por ejemplo, El documento 'nombre' afirma que: 'contenido del documento'). Esto ayuda a algunos LLM a seguir el rastro de la información.
    • Campo de filtro: Seleccione el campo de metadatos del documento que se utilizará para filtrar. Por ejemplo, puede filtrar por un campo 'tipo_de_documento' solo para tipos 'PDF'.
    • Volver a ordenar la lista de valores: Los usuarios pueden ingresar una lista priorizada de valores que desean volver a clasificar por encima de otros resultados, independientemente de la puntuación de KB. Haga clic en el icono de bombilla para agregar una nueva fila e ingresar el valor de prioridad.

    Detalles de LLM

    Aquí es donde los usuarios pueden agregar un LLM de su elección y establecer o modificar la configuración de su modelo LLM. Al hacer clic en Agregar un LLM, se les pedirá a los usuarios que seleccionen un LLM de su plataforma de elección e ingresen la información relevante, como claves de API, URL de puntos de conexión, ID de proyecto, etc. Use la flecha desplegable para habilitar los idiomas de su elección: hay un total de 56 idiomas LLM compatibles. Los usuarios también pueden modificar qué funciones de NeuralSeek se van a habilitar para el LLM agregado. Tenga en cuenta que debe agregar al menos un LLM. Si agrega varios, NeuralSeek equilibrará la carga entre ellos para las funciones seleccionadas que tienen varios LLM. Las funciones que un LLM no es capaz de no estarán disponibles para su selección. Si no proporciona un LLM para una función, no hay respaldo y esa función de NeuralSeek se deshabilitará.

    • Clave de API/acceso: La clave de API o de acceso del proveedor de LLM / Servicio.
    • Clave secreta/Zen: La clave secundaria del proveedor de servicios (solo para algunos proveedores)
    • Punto final: La URL del punto final para el servicio seleccionado.
    • Región: La región donde se aprovisiona el servicio LLM.
    • ID de proyecto: El ID de proyecto para el espacio de trabajo LLM.
    • Idiomas de LLM: Habilite o deshabilite idiomas específicos para ser utilizados con el LLM seleccionado.
    • Funciones de LLM: Habilitar o deshabilitar funciones específicas para cada LLM, lo que esencialmente selecciona qué LLM usar para cada tarea o permite el equilibrio de carga para tareas específicas. Por ejemplo, una opción es usar un LLM específico para seek y un LLM diferente para maistro o translate, lo que permite casos de uso flexibles y específicos.
    • ID de LLM: El ID/nombre interno del LLM seleccionado. Usa este ID en las llamadas a la API o desde mAIstro.
    • Prueba: Ejecuta una prueba de finalización contra el LLM, verificando las credenciales. Esto no se 'guarda', aún debes 'guardar' tu configuración con el botón principal de la interfaz de usuario.
    • Eliminar: Elimina el LLM seleccionado de la configuración.

    Nota

    Esta sección solo está disponible si estás usando el plan BYOLLM (trae tu propio LLM) de NeuralSeek.

    No todos los proveedores de LLM son iguales: se enumeran todas las opciones, aunque es posible que tu proveedor no necesite estos parámetros específicos.

    Preferencias de la empresa/organización

    Aquí es donde puedes configurar el nombre de tu empresa y la descripción de lo que se enfoca principalmente la empresa.

    • Nombre de la empresa u organización: Este campo se usa para ayudar a alinear las consultas de los usuarios con la base de conocimientos de la empresa. Por ejemplo, ¿Cómo uso tu producto? se dirigirá hacia este valor.
    • Afinidad de respuesta de la empresa: Habilitar para agregar afinidad a tu empresa o marca, además del texto existente que puede estar presente en tu base de conocimientos y discursos.
    • Discurso de campaña: Efectivamente un documento siempre anclado que se incluye en la documentación para cada llamada seek. Esto ayuda a responder preguntas como una fuente de conocimiento de respaldo cuando la búsqueda del usuario no produce documentación relevante.

    Preferencias de la plataforma

    Tu centro de operaciones para todas las preferencias relacionadas con la plataforma. Establece tiempos de espera, configura enlaces incrustados, formato de salida del Agente Virtual, etc.

    • Tiempo de espera: Establece esto a unos segundos menos que el tiempo de espera de tu plataforma de chatbot. Cuando se alcanza el tiempo de espera, NeuralSeek intentará responder con una respuesta en caché si está disponible.
    • Turnos de contexto: El número de turnos en la conversación que se pasarán al LLM. No se recomienda aumentar esto, ya que reducirá el contexto LLM disponible para tu documentación.
    • Idioma de salida predeterminado: El idioma para responder. Establecer el valor del idioma en la API anulará este parámetro. Establecer en Coincidir entrada para intentar identificar el idioma de entrada y responder en ese idioma detectado.
    • Traducir a idioma de base de conocimientos: Cuando está habilitado, traduce las consultas al idioma seleccionado en la base de conocimientos.
    • Tipo de Agente Virtual: Selecciona el tipo de Agente Virtual para la creación de respuestas. Este es el formato que NeuralSeek usará para construir el archivo del chatbot.
    • Incrustar enlaces en las respuestas devueltas: Habilitar para incrustar enlaces clicables en las respuestas generadas por seek en el lado de la API.
    • Lista de palabras vacías personalizadas: Palabras vacías - Una lista de palabras no útiles o insignificantes para eliminar antes del procesamiento. Agrega palabras aquí para anular la lista de palabras vacías de NeuralSeek.

    Categorización de intenciones

    Crea tipos de categorías y descripciones en lenguaje natural para controlar cómo se pueden categorizar las intenciones en las preguntas de los usuarios. Normalmente, una pregunta se categorizaría automáticamente como FAQ, pero puedes proporcionar categorías personalizadas adicionales aquí.

    • Categoría: El nombre de la categoría.
    • URL/Enlace: El enlace a devolver para las respuestas en esta categoría. Esto anula la URL devuelta desde la fuente de documentación.
    • Descripción: Una descripción en lenguaje natural de la categoría. Por ejemplo, raza de perro, como Yorkie o Labrador.

    Gobernanza y Salvaguardas

    Configuraciones relacionadas con la gobernanza, la protección de indicaciones, el filtrado de palabras ofensivas, etc.

    Puntuación Semántica

    Alterna los iconos para habilitar o deshabilitar el Modelo de Puntuación Semántica, usando la Puntuación Semántica como base para el Aviso y la Confianza Mínima (p. ej. NO habilitar para casos de uso que requieran traducción de idiomas), reordenando los resultados de búsqueda en función de la Coincidencia Semántica y verificando los títulos y las URL de los documentos como parte de la Coincidencia Semántica. Ten en cuenta que la puntuación semántica no será precisa cuando se obtengan respuestas en un idioma diferente al de los documentos de la base de conocimiento.

    • Habilitar el Modelo de Puntuación Semántica: El modelo de Puntuación Semántica compara la respuesta generada con las fuentes de la base de conocimiento y califica la respuesta en función de la cantidad y el enfoque de las coincidencias con la base de conocimiento y la documentación de los discursos de campaña (p. ej. ¿la respuesta se forma principalmente a partir de una sola fuente o de varias?).
    • Usar la Puntuación Semántica como base para Aviso/Mínima: Habilitar esto usa la Puntuación Semántica para los ajustes de advertencia/confianza mínima. Deshabilitar usará el porcentaje de confianza de la base de conocimiento en su lugar. No se recomienda para casos de uso que no sean en inglés.
    • Reordenar los resultados de búsqueda en función de la Coincidencia Semántica: Reordenar los fragmentos de documentación de la base de conocimiento en función de qué tan bien coincide el pasaje con la respuesta dada.
    • Verificar los títulos de los documentos como parte de la Coincidencia Semántica: Incluir el título del documento al calcular la Puntuación de Coincidencia Semántica.
    • Verificar las URL de los documentos como parte de la Coincidencia Semántica: Incluir la URL del documento al calcular la Puntuación de Coincidencia Semántica.
    • Eliminar oraciones que contengan palabras clave alucinadas: Eliminar palabras clave que no estén contenidas o relacionadas con la base de conocimiento.

    • Ajuste del Modelo Semántico: Usa las escalas deslizantes para ajustar aún más la Coincidencia Semántica.
    • Penalización por falta de términos clave de búsqueda: Después de la puntuación, se aplica esta penalización a las respuestas que carecen de atribución de la base de conocimiento de los sustantivos propios que se incluyeron en la búsqueda.
    • Penalización por falta de términos de búsqueda: Después de la puntuación, se aplica esta penalización a las respuestas que carecen de atribución de la base de conocimiento de otros sustantivos que se incluyeron en la búsqueda.
    • Penalización por salto de fuente: Cuando las respuestas se unen a través de muchos documentos fuente, puede ser una indicación de pérdida de significado o intención, dependiendo de la documentación de la fuente.
    • Peso de la cobertura total: Al mirar la respuesta, cuánto peso se debe dar a la cobertura total, independientemente de otras penalizaciones. Aumentar esto ayuda a evitar puntuaciones anormalmente bajas de respuestas largas y muy cosidas. Disminuir esto captará mejor las alucinaciones en respuestas cortas.
    • % mínimo de cobertura para reordenar: Cuál es la cobertura mínima del total de la respuesta que el documento fuente más utilizado necesita para ser reordenado sobre el documento mejor puntuado en la base de conocimiento.
    • Palabras o frases a permitir: Siempre evitar penalización para las palabras o frases seleccionadas en las respuestas.

    Mitigación de Inyección de Indicaciones

    Los usuarios pueden bloquear intentos maliciosos de usuarios que intentan hacer que el LLM responda de manera disruptiva, vergonzosa o dañina.

    • Umbral de eliminación de inyección de indicaciones: Una escala deslizante para eliminar partes de la entrada del usuario que excedan este porcentaje especificado contra el modelo de Inyección de Indicaciones, permitiendo el filtrado parcial de entradas sin bloquear toda la indicación.
    • Umbral de inyección de indicaciones: Una escala deslizante para bloquear cualquier entrada que tenga un puntaje más alto que este porcentaje contra el modelo de Inyección de Indicaciones.
    • Acción de palabra bloqueada: Eliminar las palabras ofensivas del ingreso del usuario o bloquear la pregunta por completo.
    • Lista de palabras bloqueadas: Ingrese palabras o frases (separadas por comas) que no se permiten en el ingreso del usuario. Esto es útil para bloquear nombres de clientes o productos específicos de la competencia, así como otras palabras sensibles que no están cubiertas por el corpus base de NeuralSeek.

    Información de identificación personal (PII)

    Los usuarios pueden definir cómo manejar los datos de PII detectados que se incluyeron en la pregunta.

    • Acción a tomar: Especifique las acciones cuando se detecta PII:
      • Enmascarar: Enmascarar y almacenar los datos de PII cuando se detecten en el ingreso del usuario. El enmascaramiento ocultará los datos de PII en todas las ubicaciones.
      • Marcar: Marcar y almacenar los datos de PII cuando se detecten en el ingreso del usuario. Los datos de PII se marcarán en todas las ubicaciones.
      • No hacer nada: No se tomará ninguna acción cuando se detecten datos de PII en el ingreso del usuario. Se almacenarán en texto sin formato.
      • Ocultar (retener para análisis): Ocultar (enmascarar) los datos de PII cuando se detecten en el ingreso del usuario, pero mantener los datos de PII en la base de datos para análisis.
      • Eliminar (incluyendo de análisis): Eliminar por completo los datos de PII cuando se detecten en el ingreso del usuario, incluyendo de los análisis almacenados.
    • Confiar en las palabras encontradas en los documentos de origen: Indicar si se deben reconocer u omitir ciertos términos de confianza en los documentos de origen.
    • Filtros de PII previos al LLM: Estos se ejecutan dinámicamente en el ingreso del usuario antes de enviarlo al LLM o a la base de conocimientos. Haga clic en el icono de bombilla para agregar una descripción, como un número de teléfono y una expresión regular correspondiente.
    • Filtros de PII basados en LLM: Estos utilizan el LLM elegido para identificar los datos de PII. Haga clic en el icono de bombilla para agregar una oración de ejemplo y los elementos de PII correspondientes, separados por comas.
    • Detectores de PII de NeuralSeek: Seleccione los detectores de PII predeterminados de NeuralSeek para capturar los datos de PII.

    Nota

    Utilice la función "Probar" para probar los filtros de PII establecidos. Ingrese una oración de ejemplo y haga clic en el botón "Probar". La salida mostrará la oración de prueba, una respuesta verdadera o falsa si se detectaron datos de PII y qué elemento de la oración se detectó como PII.

    Filtro de lenguaje ofensivo / Filtro de odio, abuso y lenguaje ofensivo (HAP)

    Los usuarios pueden habilitar o deshabilitar el filtro de lenguaje ofensivo, así como agregar un texto para responder con preguntas sensibles que se bloqueen.

    • Habilitar filtro de lenguaje ofensivo: Elija qué filtro utilizar para el filtrado de lenguaje ofensivo. Puede utilizar el punto final de moderación del LLM si está disponible, el filtro de NeuralSeek o desactivarlo.
    • Texto de respuesta bloqueado: El texto que se mostrará cuando se bloquee la entrada o la pregunta. Por ejemplo, Esa parece ser una pregunta delicada.

    Protección de atribución

    • Ajuste la tolerancia a la desinformación para generar texto sobre la empresa o asociar personas u objetos que carecen de referencias específicas en el material de la base de conocimientos utilizando la escala deslizante de Rígido a Estándar. Cuanto más rígidos sean sus ajustes, mayores serán las posibilidades de bloquear ocasionalmente preguntas legítimas que utilicen un vocabulario alternativo o que estén mal documentadas en su base de conocimientos.

    Confianza de advertencia

    Utilice la escala deslizante para aumentar el umbral de confianza.

    • Porcentaje de confianza: Cualquier respuesta inferior a este número tendrá el siguiente texto antepuesto a la respuesta dada.
    • Texto de advertencia: El texto que se antepondrá a la respuesta dada.

    Confianza mínima

    Utilice la escala deslizante para aumentar el porcentaje de confianza mínimo y el porcentaje de confianza mínimo para mostrar una URL. Agregue un texto para responder con preguntas que no cumplan con la confianza mínima y seleccione si desea agregar una URL de respaldo en el mínimo. (por ejemplo, no hay nada en nuestra base de conocimientos sobre eso).

    • Porcentaje de confianza mínimo: Cualquier respuesta inferior a este número tendrá el texto a continuación sustituido en lugar de la respuesta.
    • Plantilla mAIstro Opcional - Una plantilla mAIstro para manejar la confianza mínima de una manera personalizada.
    • Texto de respuesta: La respuesta a dar cuando las respuestas están por debajo del porcentaje de confianza mínimo establecido.
    • Porcentaje de confianza mínimo para mostrar una URL: Cualquier respuesta inferior a este número no devolverá una URL vinculada.
    • URL de respaldo Opcional - Una URL para ofrecer cuando no se cumple la confianza mínima.

    Confianza mínima con el plan de respaldo de mAIstro

    Utilice la plantilla de mAIstro para establecer un plan de respaldo para los casos en los que no se cumpla el umbral de confianza. Esto asegura una transición sin problemas a una respuesta o acción más adecuada, como notificar a los equipos o escalar el problema.

    1. Cree una plantilla de mAIstro: Cree una plantilla de respaldo para manejar escenarios de baja confianza. En el nodo 'Confianza mínima - ENTRADA' (que debe estar ubicado en la parte superior), defina la lógica de cómo debe responder.

    1. Seleccione su plantilla: Una vez que su plantilla esté lista, selecciónela en el menú desplegable 'Plantilla de mAIstro para el mensaje de confianza mínima personalizado'.

    1. Pruebe con consultas fuera del ámbito: Después de la configuración, intente hacer una pregunta que esté fuera de la base de conocimientos. Seek recurrirá a su nueva plantilla. ¡Juegue con ella y vea cómo maneja los escenarios de respaldo!

    • Notificar a través de Slack: Si se hace una pregunta fuera del ámbito, notifique a su equipo en Slack para que puedan mejorar la documentación para su uso futuro.

    • Crear un problema en GitHub: Cree automáticamente un problema de GitHub con detalles como minConfMsg.originalQuery y minConfMsg.language.

    Texto mínimo

    Utilice la escala deslizante para establecer una cantidad mínima deseada de palabras en una pregunta.

    • Palabras mínimas: El número mínimo de palabras en una pregunta/entrada del usuario.

    • Plantilla de mAIstro Opcional - Una plantilla de mAIstro para manejar el texto mínimo de una manera personalizada. En este caso, debemos usar el nodo 'Texto mínimo - ENTRADA' y puede probar flujos de plantillas como en la sección 'Confianza mínima'.

    • Texto de respuesta: Agregue un texto para responder con preguntas que no cumplan con la longitud mínima de la entrada de palabras.

    Longitud máxima

    Utilice la escala deslizante para establecer una cantidad máxima deseada de palabras en una pregunta.

    • Palabras máximas: El número máximo de palabras en una pregunta/entrada del usuario. Establézcalo en 100 para eliminar el límite. Utilice un límite bajo para ayudar a mitigar las preguntas adversarias diseñadas para generar respuestas inapropiadas.

    • Plantilla de mAIstro Opcional - Una plantilla de mAIstro para manejar el texto máximo de una manera personalizada. En este caso, debemos usar el nodo 'Máximas palabras - ENTRADA' y puede probar flujos de plantillas como en la sección 'Confianza mínima'.

    • Texto de respuesta: Agregue un texto para responder con preguntas que superen el límite de palabras de entrada.

    Configuración avanzada

    Estas opciones (ampliadas) están disponibles después de habilitar Mostrar opciones avanzadas desde la configuración predeterminada.

    Ajuste de la base de conocimientos

    Ajustar su base de conocimientos es una parte importante de crear un sistema que funcione bien.

    • Rango de puntuación de documentos: El rango de puntuación superior de los documentos a devolver. Por ejemplo, cuando se establece en 0.8 o 80%, devuelve el 80% superior de los documentos con mayor puntuación, descartando los documentos con la puntuación más baja del 20%. Cuanto menor sea el número, más estricto será el umbral de puntuación. Generalmente es mejor establecerlo en un número alto, utilizado con la configuración de Máx. Docs.
    • Penalización por fecha del documento: Penalizar las puntuaciones de los documentos que incluyen fechas antiguas. Un número más alto significa una penalización más alta para los documentos más antiguos, escalando con el tiempo/la antigüedad.
    • Caché de consultas de KB: Limitar las consultas repetidas a la base de conocimientos almacenando en caché las consultas de KB. Establece esto en el número de minutos que deseas conservar las consultas de KB almacenadas en caché.
    • Máx. documentos por búsqueda: El número de documentos que se enviarán al LLM en cada acción de búsqueda. Generalmente los mejores resultados se obtienen con este valor establecido en 4-5 documentos.
    • Tamaño del fragmento: El recuento de caracteres que se pasará a la KB para el tamaño del pasaje del documento. Cuanto mayor sea el número, más grande será el fragmento de documentación. Generalmente es mejor que sea un número más pequeño, alrededor de 500.
    • Puntuación máxima bruta: La puntuación de documento más alta de todos los tiempos que NeuralSeek ha visto de la KB. NeuralSeek usa este número internamente para calcular una puntuación del 100% para los documentos.

    Configuración de búsqueda híbrida y vectorial

    Ofrecer la posibilidad de elegir entre buscar con Lucene, Vector o un enfoque híbrido

    • Tipo de consulta: El tipo de consulta que NeuralSeek utilizará para recopilar la documentación de origen.
    • Lucene: Consultas de coincidencia exacta.
    • Vector: Búsqueda de similitud vectorial, basada en el modelo Vector implementado.
    • Híbrido: Una consulta de búsqueda combinada que aumenta ligeramente los resultados de Lucene, permitiendo una recuperación gradual a los resultados de Vector si no se encuentran coincidencias exactas.
    • ¿Usar el modelo ELSER de Elastic? El formato de consulta es diferente al usar ELSER vs. los modelos KNN implementados. Selecciona Falso si no estás usando el modelo ELSER de Elastic.
    • ELSER - ID del modelo: El nombre del modelo implementado y en ejecución.
    • ELSER - Campo de incrustación: El nombre del campo de metadatos donde se almacenan las incrustaciones vectoriales generadas.
    • Consulta KNN de Elastic: El JSON de la consulta de Vector KNN que se ejecutará. Hay un par de valores que se deben establecer dentro del JSON:
    • field: El nombre del campo de metadatos donde se almacenan las incrustaciones vectoriales.
    • model_id: El nombre del modelo implementado y en ejecución.
    • model_text: Ofrecemos una variable de expansión << query >> para insertar la consulta generada por NeuralSeek. Útil para editar si algunos modelos vectoriales requieren un formato específico, p. ej. question: <<query>>
    • Consulta la documentación de Elastic para obtener más información sobre los otros parámetros disponibles.

    Ingeniería de prompts

    Esto permite a los usuarios expertos inyectar instrucciones específicas en el prompt base del LLM. La mayoría de los casos de uso no necesitarán esto y no deberían usarlo. (por ejemplo, no ingreses proporcionar información fáctica o actuar como un agente de atención al cliente útil. El extenso prompt de NeuralSeek ya hace esto).

    No habilites esto a la ligera, ya que puedes debilitar las salvaguardas y el extenso prompt que NeuralSeek proporciona de forma predeterminada. No uses ningún idioma que no sea inglés en la ingeniería de prompts.

    • Instrucciones del prompt: Texto para agregar al final del prompt del LLM. Esto rara vez puede ser útil, pero algunos ejemplos podrían ser Responder con una lista con viñetas si es posible, o Responder en dialecto de vaquero.

    Respuesta Ingeniería y Preferencias

    La personalización de la ingeniería de respuestas y el establecimiento de preferencias proporciona adaptabilidad a diferentes contextos.

    • Verbosidad de la respuesta Utilice las escalas deslizantes para establecer si la generación de respuestas se ceñirá a ser concisa y breve, o ofrecerá más libertad para ser flexible y verbosa.
    • Forzar respuestas de la base de conocimientos: Habilitar para agregar un prompt extra para ayudar a forzar las respuestas de la documentación devuelta. En general, es mejor mantener esto habilitado.
    • Expresiones regulares: Haga clic en el icono de bombilla para agregar una nueva fila. Ingrese una expresión regular y un reemplazo correspondiente. Por ejemplo, use esta función para eliminar o intercambiar números de teléfono, correos electrónicos, etc.

    Coincidencia de intención y configuración de caché

    NeuralSeek genera y agrupa automáticamente las entradas de los usuarios en intenciones. Cuando una entrada de usuario no coincide con una intención existente, la pregunta se agrega al grupo de FAQ genérico.

    Los siguientes tipos de coincidencias de intención están disponibles:

    • Coincidencia exacta: La entrada del usuario coincide exactamente con una intención.
    • Similitud vectorial: Compare la similitud vectorial de la entrada del usuario con las intenciones existentes, coincidiendo con intenciones similares.
    • Utilice la función "Pruébalo" para probar la similitud de intención. Ingrese una oración de ejemplo y haga clic en el botón "Probar". La salida mostrará la intención similar extraída de la pestaña "Curar" y un puntaje de similitud correspondiente.
    • Utilice la escala deslizante "Umbral de coincidencia de intención" para establecer el porcentaje mínimo de coincidencia para hacer coincidir una Intención existente.
    • Coincidencia difusa: La entrada del usuario coincide estrechamente con una intención, pero no exactamente.
    • Coincidencia de palabras clave: La entrada del usuario contiene palabras clave que coinciden exactamente con las palabras clave de una intención.
    • Coincidencia difusa de palabras clave: La entrada del usuario contiene palabras clave que coinciden estrechamente con una intención.

    Los usuarios también pueden configurar cómo se debe realizar el almacenamiento en caché de respuestas para respuestas editadas y normales. Esto es útil para acelerar los tiempos de respuesta y producir resultados más consistentes.

    • Caché de respuestas editadas: Defina la cantidad mínima de respuestas editadas antes de que se detenga la generación de lenguaje y se sirvan las respuestas almacenadas en caché. Establezca la escala en '0' para deshabilitar la caché de respuestas editadas.
    • Caché de respuestas normales: Defina la cantidad mínima de respuestas generadas normalmente para almacenar en caché antes de que se detenga la generación de lenguaje y se sirvan las respuestas almacenadas en caché. Establezca la escala en '0' para deshabilitar la caché de respuestas editadas.

    Nota

    Las respuestas editadas tienen prioridad en la Caché de respuestas normales, seguidas de la respuesta generada más reciente.

    Comprensión de tablas

    Esto preprocesa sus documentos para extraer y analizar datos tabulares en un formato adecuado para consultas conversacionales. Dado que este proceso de preparación es costoso y consume mucho tiempo, esta función es de opción y consumirá 1 consulta de búsqueda por cada tabla preprocesada. Las colecciones de rastreo web no son elegibles para la comprensión de tablas, ya que el intervalo de nuevo rastreo provocará un uso excesivo de recursos informáticos. El tiempo de preparación de la tabla tarda varios minutos por página. Comuníquese con cloud@cerebralblue.com con los detalles de su oportunidad y caso de uso para que se le considere para el acceso.

    • IDs de colección de descubrimiento: Haga clic en el icono de bombilla para agregar una nueva fila. Ingrese el ID de colección deseado de Watson Discovery para la preparación de tablas.

    Nota

    La comprensión de tablas requiere un LLM compatible con la función de comprensión de tablas habilitada. No todos los LLM son capaces de la comprensión de tablas.

    Filtro de documentos corporativos

    Conecte NeuralSeek a un motor de reglas corporativas externo para filtrar la documentación permitida por usuario. Cada solicitud enviará los ID de la documentación encontrada a un punto final que usted configure aquí. Cualquier ID que no se devuelva del filtro corporativo se bloqueará.

    • Habilitar filtro corporativo: Si está habilitado, complete toda la información relevante, incluyendo:
    • URL base para el filtro corporativo (obtener): La URL del motor de filtro de documentos corporativos.
    • Parámetro de URL para el nombre de usuario: El nombre del parámetro para la identificación del usuario.
    • Parámetro de URL para el campo KB: El nombre del parámetro para el ID del documento para el filtrado de permisos.
    • Campo de base de conocimiento a enviar: El campo de metadatos de KB para enviar como ID del documento.

    Registro corporativo

    Conecte NeuralSeek a un punto final de registro de auditoría corporativa. Cuando esté conectado y habilitado, todas las solicitudes y respuestas al punto final de la API de Seek, así como la pestaña Curate, se registrarán en su instancia de Elasticsearch. Los usuarios pueden registrar el aviso completo de LLM Seek para fines de auditoría y cumplimiento.

    • Habilitar registro corporativo: Alterne el icono para habilitar o deshabilitar esta función. Si está habilitado, complete toda la información relevante, incluidos el punto final de Elasticsearch y la clave de API de Elasticsearch.
    • Registro de avisos: Escriba "acepto" en el cuadro proporcionado para aceptar el Acuerdo de no divulgación proporcionado y habilitar el registro de avisos.
