    Control de flujo

    Llamar a otra plantilla

    {{ maistro|template: nombreDePlantilla }}

    Importa el contenido de nombreDePlantilla en el entorno actual.

    Nota

    Dada una plantilla de ejemplo, actualizaciones_neuralseek:

    Basado en los registros de cambios que se encuentran aquí:
{{ web|url:https://documentation.neuralseek.com/changelog/ }}
enumera los elementos del último mes.
{{ LLM }}

    Simplemente usando {{ maistro|template: actualizaciones_neuralseek }} producirá el resultado de ejemplo.

    Nota

    Para pasar parámetros a las plantillas de mAIstro, simplemente define las variables en tu entorno actual.

    Dada una plantilla de ejemplo, actualizaciones_neuralseek:

    Basado en los registros de cambios que se encuentran aquí:
{{ web|url:<< name:'url' >> }}
enumera los elementos del último mes.
{{ LLM }}

    Para pasar la variable url a la plantilla:

    {{ variable  | name: url | value: https://documentation.neuralseek.com/changelog/ }}

{{ maistro|template: actualizaciones_neuralseek }}

    Esto permite que las plantillas se incorporen al contexto actual, efectivamente empalman el contenido en el contexto deseado.

    Establecer variable

    Crea o establece una variable que se puede usar más adelante en la expresión NTL. Por ejemplo, 

    34=>{{ variable | name:edad }}
o
{{ variable  | name: edad | value: 34 }}

    Ejemplo

    • Nombre: El nombre de la variable a establecer.
    • Valor: El valor opcional (de reemplazo) para establecer en la variable.

    Éxito

    Usar variable

    Sintaxis para usar / expandir una variable en el entorno.

    << name: nombreDeVariable, prompt: true >>

    Ejemplo

    • Nombre: El nombre de la variable
    • Prompt: Si se establece en true, la interfaz de usuario solicitará el valor de esta variable. Si se establece en false, la interfaz de usuario evitará solicitar este variable.

    Éxito

    • El contenido de la variable.

    Nota

    Cuando la variable NO se encuentra pero se usa en la notación << >>, la variable se considera como entrada del usuario, y mAIstro solicitará el valor antes de la evaluación. Por ejemplo, si tienes << name: nuevo >> o << name: nuevo, prompt: true >> y no hay tal cosa como {{ variable|name: nuevo }} en la expresión, mAIstro lo pedirá así:

    Detener

    Detiene todo procesamiento adicional. Punto final.

    {{ stop }}

    Bucles: Inicio, Fin, Romper

    Inicio del bucle

    Denota el inicio de un bucle y declara el número máximo de bucles a realizar.

    {{ startLoop  | count: 3 }}

    Fin del bucle

    Denota el final de un bucle. Esto no detiene el bucle, sino que envía la ejecución de vuelta al principio si aún no se ha alcanzado el número total de bucles.

    {{ endLoop }}

    Romper el bucle

    Detiene un bucle de forma anticipada. Útil con el nodo de condición.

    {{ breakLoop }}
    Nota 
    {{ variable  | name: count | mode:  | value: 0 }}
{{ startLoop  | count: 5 }}
{{ math  | equation: << name: count >> + 1 }}=>{{ variable  | name: count }}
{{ endLoop  }}
El recuento ahora es: << name: count >>

    Producirá:

    El recuento ahora es: 6

    Nota

    El número de bucles asignado en startLoop es el número de veces adicionales que se ejecutarán los nodos. Como se ve arriba, el nodo del medio (math) se ejecutará un total de 6 veces: una vez para comenzar y luego 5 veces más (el número de bucles establecido).

    Bucle de variables

    La función de Bucle de Variables nos permite recorrer matrices, matrices anidadas u objetos JSON. Puede usar esta función para dar formato a los arrays de manera ordenada o para realizar acciones sobre el contenido durante cada bucle.

    {{ variableLoop | variable: categories | loopType: array-strings }}

    Nota 
    {looper: [a,b,c]}=>{{ jsonToVars }}
{{ variableLoop | variable: looper | loopType: }}
fue la entrada proporcionada.
{{ variable | name: myVar | mode: append }}
{{ endLoop }}
<< name: myVar, prompt: false >>

    Dará como resultado:

    a
fue la entrada proporcionada.
b
fue la entrada proporcionada.
c
fue la entrada proporcionada.

[2023-04-12T10:30:15Z] Usuario: Estoy bien, gracias por preguntar. ¿Cómo puedo ayudarte?
[2023-04-12T10:30:30Z] Asistente: Me temo que no tengo una tarea específica para ti en este momento. Simplemente estoy aquí para charlar y ayudar en lo que pueda.
[2023-04-12T10:30:45Z] Usuario: Eso es genial, aprecio tu disponibilidad. Me preguntaba si podrías ayudarme con un proyecto en el que estoy trabajando.
[2023-04-12T10:31:00Z] Asistente: Absolutamente, estaré encantado de ayudarte con tu proyecto. Por favor, proporcióname los detalles y haré todo lo posible por ayudarte.
    • CONTIENE: Puede buscar subcadenas: CONTIENE('cadena larga para verificar', 'cadena')
    • LONGITUD: Puede evaluar la longitud de una cadena para usarla en condiciones: LONGITUD('cadena') (este ejemplo evalúa a 6)
    • Las variables deben estar envueltas en comillas simples para la comparación o la verificación de subcadenas.

    Éxito

    No hay devoluciones, sin embargo:

    • Una condición que evalúe a 'verdadero' continuará la cadena horizontal.
    • Una condición que evalúe a 'falso' detendrá la ejecución de la cadena horizontal y continuará al siguiente paso de flujo.
    Abstracto

    Ejemplo conjunto 1: Básico

    {{ condition  | value: 1 == 1 }}=>¡Esto es verdadero!

    Producirá el texto de salida: ¡Esto es verdadero!

    {{ condition  | value: (5 + 5) == 10 }}=>¡Esto es verdadero!

    Producirá el texto de salida: ¡Esto es verdadero!

    Ejemplo conjunto 2: O, Y

    {{ condition  | value: O(1==1, 2==3, 1==2, 1==1) }}=>¡Esto es verdadero!

    Continuará la cadena y producirá el texto de salida: ¡Esto es verdadero!.

    {{ condition  | value: Y(1==1, 2==2, 3==3, 4==4) }}=>¡Esto es verdadero!

    Continuará la cadena y producirá el texto de salida: ¡Esto es verdadero!.

    {{ condition  | value: O(1==2,2==3) }}=>¡Esto es verdadero!

    Detendrá la cadena y no producirá ninguna salida, ya que la cadena se bloqueó con una condición falsa.

    Ejemplo conjunto 3: Cadenas

    {{ condition  | value: 'nombre' == 'nombre' }}=>¡Esto es verdadero!

    Producirá el texto de salida: ¡Esto es verdadero!

    {{ condition  | value: CONTIENE('esta es una cadena de prueba', 'prueba') }}=>¡Esto es verdadero!

    {{ condition  | value: CONTIENE('<< name: variableContainingTest, prompt: false >>', 'prueba') }}=>¡Esto es verdadero!

    Ambos producirán el texto de salida: ¡Esto es verdadero!

    {{ condition  | value: LONGITUD('Hola Mundo') > 5 }}=>¡Esto es verdadero!

    Producirá el texto de salida: ¡Esto es verdadero!

    {{ condition  | value: (LONGITUD('<< name: variableContainingTest, prompt: false >>') + 12) > 10 }}=>¡Esto es verdadero!

    Producirá el texto de salida: ¡Esto es verdadero!
