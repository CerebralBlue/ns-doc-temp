    Configuração do ElasticSearch para Busca Vetorial

    Visão Geral

    Este guia fornece instruções passo a passo sobre como configurar a pesquisa vetorial com o ElasticSearch. Ele inclui o login nos ambientes, a criação de chaves para acesso à API, a configuração de uma instância de aprendizado de máquina, o download dos modelos necessários, a criação de índices de origem e destino e a ingestão de dados para gerar incorporações de texto. O guia também aborda as etapas de carregamento manual de dados e a utilização de funções auxiliares do cliente para a ingestão de dados. Ele conclui com a verificação dos dados e das incorporações de conteúdo no índice de destino.

    Fazer login nos Ambientes

    Comece fazendo login na sua conta do IBM Cloud

    • Para provisionar no IBM Cloud:
      • Navegue até Databases for ElasticSearch.
      • Selecione a Edição de Banco de Dados Platinum.
    • Caso contrário, provisione normalmente na Elastic Cloud.

    Existem dois ambientes para trabalhar.

    • Console do ElasticSearch Cloud. Observe os ícones no canto superior direito.
    • Console do Kibana
      • Os usuários podem ser levados diretamente para o console do Kibana após criar um deployment. Caso contrário, navegue até lá selecionando Abrir na página de deployment do console do ElasticSearch Cloud.

    Criando Chaves

    • Selecione o ícone do círculo no canto superior direito da tela do Kibana.
    • Selecione Detalhes da Conexão
    • Aqui, você verá o Endpoint do ElasticSearch e o ID da Nuvem.
    • Selecione Criar e Gerenciar Chaves de API.
    • Para criar uma nova chave de API, clique em Criar Chave de API.
      • Adicione um nome exclusivo.
      • Selecione o tipo como Chave de API de Usuário.
      • Clique no botão Criar Chave de API na parte inferior da caixa de diálogo.

    Salve esses valores em um local seguro para uso posterior.

    Criar uma Instância de Aprendizado de Máquina

    O Elastic requer uma instância de aprendizado de máquina para executar os modelos de PLN necessários para vetorizar os dados para indexação.

    • Navegue até a tela Inicial da sua instância do ElasticSearch.
    • Navegue até o deployment recém-criado e selecione Gerenciar.
    • No menu lateral, selecione Editar.
    • Role para baixo até a seção Instâncias de Aprendizado de Máquina.
    • Selecione Adicionar Capacidade.
    • Selecione 4 GB de RAM.
    • Clique em Salvar na parte inferior da página.

    Baixar Modelos

    Abstract
    • No Kibana, clique no ícone de menu no canto superior esquerdo e navegue até Analytics > Machine Learning > Trained Models.
    • Clique no botão Download na coluna Ações
      • Escolha o modelo recomendado .elser_model_2_linux-x86_64
      • Pode levar algum tempo para que o download seja concluído.
    • Clique no link Implantar que aparece quando o mouse é passado sobre o modelo baixado.
    • Deixe as configurações padrão na coluna Diálogo e selecione Iniciar.
    • A coluna Estado mostrará Implantado quando concluído com sucesso.

    Abstract

    É recomendado usar o Eland para carregar e baixar o modelo desejado no ElasticSearch.

    • Execute este comando para instalar o cliente Python Eland com PyTorch: python -m pip install 'eland[pytorch]'
    • Execute este script para baixar o modelo do Hugging Face, convertê-lo para o formato TorchScript e carregá-lo no cluster do Elasticsearch:

    ``` eland_import_hub_model

    --cloud-id \ -u -p \ --hub-model-id elastic distilbert-base-cased-finetuned-conll03-english \ --task-type ner ```

    - Especifique o **identificador da Elastic Cloud** usando a configuração TLS com um certificado baixado do **IBM Cloud -> Database for Elasticsearch -> guia Visão geral**.
- Forneça detalhes de autenticação para acessar seu cluster.
- Especifique o **identificador** do modelo no Hugging Face model hub.
- Especifique o **tipo de tarefa NLP** como `text_embedding`.

> É recomendado usar o modelo `intflost/multilingual-e5-base` do Hugging Face para começar.

> Pode levar algum tempo para o modelo iniciar automaticamente, até algumas horas.

    Criar Índice de Origem e Carregar Dados

    Os índices podem ser criados carregando manualmente os dados usando a API _bulk, ou usando uma função auxiliar do cliente que criará o índice e carregará os dados.

    Abstract
    • Navegue até o console do Kibana.
    • No menu lateral, selecione Management > Dev Tools para abrir o console de desenvolvimento.
    • Exclua qualquer código que apareça.
    • Para criar o índice de origem, insira o seguinte código:
        PUT /search-gs-docs-src
    {
    mappings: {
        properties: {
        title: { 
            type: text 
        },
        content: { 
            type: text 
        },
        source: { 
            type: text 
        },
        url: { 
            type: text 
        },
        public_record: { 
            type: boolean 
        }
        }
    }
    }
    • Clique no ícone executar.
    • Prepare os dados para ingestão em massa convertendo-os manualmente e use o console de desenvolvimento para carregá-los inserindo o seguinte código:
        POST _bulk
    { index : { _index : search-gs-docs-src, _id : 1 } }
    { title : Top 3 Best Practices to Secure Your Gainsight PX Subscription,
    content : We should all protect what has been entrusted…",
    url : https://support.gainsight.com/...,
    source : docs",
    "public_record":true,
    "objectID": "https://support.gainsight.com/..."
    }
    { index : { _index : search-gs-docs-src, _id : 2 } }
    { title : Using PX with Content Security Policy,
    content : This article describes the steps to allow a Content Security Policy…",
    url : https://support.gainsight.com/...,
    source : docs",
    "public_record":true,
    "objectID": "https://support.gainsight.com/..."
    }
    …
    Abstract
    • Insira o seguinte código para utilizar a função auxiliar do cliente para criar o índice e carregar os dados:

    ``` 'use strict'

    require('array.prototype.flatmap').shim()
const { Client } = require('@elastic/elasticsearch')
const client = new Client({
cloud: { id: '<cloud_id>'},
auth: { apiKey: '<api_key>' }
})
const dataset = require('./gainsight_documentation_data/gainsight-en-federated.json')

// Criar e carregar o índice de origem
async function run () {
await client.indices.create({
    index: 'search-gs-docs-src',
    operations: {
    mappings: {
        properties: {
        title: { type: 'text' },
        content: { type: 'text' },
        url: { type: 'text' },
        source: { type: 'text' },

    public_record: { type: 'boolean' }, objectID: { type: 'text' } } } } }, { ignore: [400] })

        const operations = dataset.flatMap(doc => [{ index: { _index: 'search-gs-docs-src' } }, doc])

    const bulkResponse = await client.bulk({ refresh: true, operations })

    if (bulkResponse.errors) {
        const erroredDocuments = []
        // The items array has the same order of the dataset we just indexed.
        // The presence of the `error` key indicates that the operation
        // that we did for the document has failed.
        bulkResponse.items.forEach((action, i) => {
        const operation = Object.keys(action)[0]
        if (action[operation].error) {
            erroredDocuments.push({
            // If the status is 429 it means that you can retry the document,
            // otherwise it's very likely a mapping error, and you should
            // fix the document before to try it again.
            status: action[operation].status,
            error: action[operation].error,
            operation: operations[i * 2],
            document: operations[i * 2 + 1]
            })
        }
        })
        console.log(erroredDocuments)
    }

    const count = await client.count({ index: 'search-gs-docs-src' })
    console.log(count)
    }

    run().catch(console.log)
    • Para obter o nome do pipeline com o modelo carregado, navegue até Kibana > Machine Learning > Modelos Treinados.
    • Expanda o modelo implantado.
    • Navegue até a guia Pipelines para visualizar o my-content-embesddings-pipeline criado na etapa acima.

    Para confirmar que a tarefa foi executada com sucesso, execute o seguinte comando usando o ID da tarefa produzido na resposta do comando anterior. GET _tasks/<task_id>.

    • Verifique se os embeddings de conteúdo estão no novo índice de destino.
      • Navegue até o Kibana.
      • Navegue até Pesquisa > Conteúdo > Índices.
      • Abra o índice search-gs-docs-dest.
      • Abra a guia Documentos para ver os dados.

    Mapear um Campo

    Modelos compatíveis com o ElasticSearch NLP geram vetores densos como saída, portanto, o tipo de campo dense_vector para o índice é adequado para armazenar. Esse tipo de campo deve ser configurado com o mesmo número de dimensões usando a opção dims.

    • Digite o seguinte código no console de desenvolvimento para criar um mapeamento de índice que define o campo contendo a saída do modelo. 

          PUT my-index
    {
    mappings: {
        properties: {
        my_embeddings.predicted_value: { 
            type: dense_vector, 
            dims: 384 
        },
        my_text_field: { 
            type: text 
        }
        }
    }
    }

    • my_embeddings.predicted_value é igual ao nome do campo contendo os embeddings gerados pelo modelo.

    • O campo type deve ser dense_vector.
    • O campo dims contém o número de dimensões dos embeddings produzidos pelo modelo. Certifique-se de que esse número esteja configurado no campo dense_vector.
    • O campo my_text_field é igual ao nome do campo a partir do qual criar a representação de vetor denso.
    • O campo type é text.

    Testar a Pesquisa Semântica

    Abstract

    Teste a pesquisa semântica usando a consulta text_expansion fornecendo o texto da consulta e o ID do Modelo ELSER.

    • Digite o seguinte código no console de desenvolvimento:
            GET search-gs-docs-dest/_search
    {
    query:{
        text_expansion:{
            content_embedding:{
                model_id:.elser_model_2_linux-x86_64,
                model_text:Insira a consulta de amostra aqui
            }
        }
    }
    }
    • O campo content_embedding contém a saída ELSER gerada.
    Abstract

    Os modelos de vetor denso permitem que os usuários consultem recursos de classificação com uma pesquisa kNN. No cláusula knn, os usuários fornecerão o nome do campo de vetor denso. Na cláusula query_vector_builder, adicione o ID do modelo e o texto da consulta.

    • Digite o seguinte código no console de desenvolvimento:
        GET my-index/_search
    {
    knn: {
        field: my_embeddings.predicted_value,
        k: 10,
        num_candidates: 100,
        query_vector_builder: {
        text_embedding: {
            model_id: sentence-transformers__msmarco-minilm-l-12-v3,
            model_text: a string de consulta
        }
        }
    }
    }

    Conectar o NeuralSeek ao Elasticsearch

    • Navegue até sua conta no IBM Cloud.
    • Abra a instância de serviço do NeuralSeek.
    • Navegue até a tela Configurar.
    • Salve suas configurações atuais clicando no botão Baixar Configurações na parte inferior da tela.
    • Abra o accordion Conexão com o KnowledgeBase e atualize os seguintes campos.
      • Defina o Tipo de KnowledgeBase como ElasticSeach
      • Defina o Endpoint do ElasticSearch.
      • Defina a Chave de API Privada do ElasticSearch.
    • Defina o Nome do Índice do ElasticSearch para o índice de destino. Neste caso, search-gs-docs-dest.
      • Defina o Campo de Dados de Curadoria como content.
      • Defina o Campo de Nome da Documentação como title.
      • Defina o Campo de Link como url.
    • Clique no botão Salvar na parte inferior da página.

    Ativar a Pesquisa Vetorial no NeuralSeek

    Na tela de Configuração do NeuralSeek, abra o accordion Configurações de Pesquisa Híbrida e Vetorial para atualizar os seguintes campos.

    • Defina o Tipo de Consulta Elástica como Híbrido.
      • Isso permitirá tanto a pesquisa Lucene (correspondência exata) quanto a pesquisa Vetorial (semântica) para alcançar uma resposta mais robusta.
    • Defina o ID do Modelo como .elser_model_2_linux-x86_64
    • Defina o Campo de Incorporação como content_embedding
    • Defina o campo Usar o Modelo ELSER Elástico como True para usar o Modelo ELSER, ou defina como False para permitir que o NeuralSeek espere o formato JSON para uma consulta kNN.
    • Clique em Salvar na parte inferior da tela.

    !!! warning Se estiver usando 'Bancos de Dados IBM para ElasticSearch'

    Com a pesquisa híbrida, a KnnScoreDocQuery foi criada por um leitor diferente. Para corrigir isso, insira o seguinte código no console de desenvolvimento do Kibana:
```
    PUT /<INDEX_NAME>/_settings
    {
        index : {
            highlight.weight_matches_mode.enabled : false
        }
    }
```
