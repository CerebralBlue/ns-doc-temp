    Editor visual

    Descripción general

    Presentando mAIstro: un espacio de juego abierto para modelos de lenguaje grande, diseñado para facilitar el tiempo y el esfuerzo de desarrollo.

    mAIstro es una herramienta práctica que le brinda las siguientes capacidades:

    1. Elección de LLM: (Planes BYOLLM) Seleccione su LLM preferido e intégrelo sin problemas con mAIstro.
    2. Utilizar el lenguaje de plantillas NeuralSeek (NTL): Cree indicaciones dinámicas usando una combinación de palabras regulares y marcado NTL para recuperar contenido de diferentes fuentes.
    3. Editor visual amigable: Cree indicaciones personalizadas con un editor visual fácil de usar.
    4. Utilizar otras funciones de NeuralSeek: Extraer, proteger o buscar una consulta a través de la plataforma mAIstro.
    5. Recuperación de contenido versátil: Recupere datos de varias fuentes, incluidas bases de conocimiento, bases de datos SQL, sitios web, archivos locales o su propio texto.
    6. Mejora del contenido: Mejore sus datos con funciones como resumen, eliminación de palabras vacías, extracción de palabras clave y eliminación de PII para garantizar que su contenido esté refinado y sea valioso.
    7. Indicaciones protegidas: mAIstro proporciona protección contra inyección de indicaciones y salvaguardas de palabras ofensivas, evitando momentos embarazosos con la generación de lenguaje.
    8. Comprensión de tablas: Realice búsquedas y genere respuestas con consultas en lenguaje natural contra datos estructurados.
    9. Salida sin esfuerzo: Vea fácilmente el contenido generado dentro del editor incorporado o expórtelo directamente a un documento de Word, lo que ofrece un control conveniente sobre su salida.
    10. Puntuación semántica de precisión: Además, todas estas operaciones se evalúan utilizando nuestro modelo de puntuación semántica. Esto permite obtener información sobre el alcance del contenido adaptado a sus preferencias.

    Editor visual

    • El Editor visual permite a los usuarios crear expresiones utilizando bloques movibles, encadenados y personalizables que ejecutan comandos. Simplifica la interacción del usuario a través de bloques de arrastrar y soltar, lo que facilita la navegación por casos de uso complejos sin necesidad de código.

    Editor NTL

    • El Editor NTL permite a los usuarios avanzados o desarrolladores crear expresiones utilizando NTL Markdown. Esto muestra el NTL Markdown sin procesar, lo que le permite editar manualmente o copiar toda la plantilla para compartir y depurar.

    Inspector de mAIstro

    • El Inspector de mAIstro (el pequeño icono de insecto cerca de la esquina superior derecha) permite a los usuarios profundizar en los detalles de cada paso, exponiendo qué se estableció, cuándo se estableció y cómo se procesó.
    • Expanda los pasos individualmente para profundizar en valores, cálculos, asignaciones o generación específicos.

    Inicio rápido con el constructor automático

    Comience dando una indicación en el constructor automático. Use este ejemplo de indicación: Construir una plantilla para enviar correos electrónicos personalizados a cada dirección enumerada en un archivo CSV de entrada. Esto le da la capacidad de comenzar desde cero, usar una plantilla existente o construir una usando comandos en lenguaje natural.

    Esto generará una plantilla personalizable que puede probar o adaptar a sus necesidades.

    Comprender el Editor visual

    Hacer clic para insertar

    Todos los elementos del panel izquierdo se pueden crear en el editor haciendo clic en ellos.

    Hacer clic para editar

    Seleccionar una tarjeta resaltará el nodo de color azul y aparecerá un cuadro de diálogo en el lado derecho para editar las opciones de configuración del nodo seleccionado. Dependiendo del tipo de nodo, puede haber varias opciones. Consulte la página de referencia de NTL para obtener una descripción de todas las opciones configurables.

    Eliminar un nodo

    Puede eliminar un nodo haciendo clic en el botón rojo Eliminar nodo en la parte inferior del panel de opciones.

    Menús emergentes

    Los menús emergentes permiten a los usuarios acceder e insertar fácilmente secretos, variables definidas por el usuario, variables definidas por el sistema o generar nuevas variables mientras trabajan en el constructor visual. Esta función mejora el proceso de construcción al proporcionar un acceso rápido a elementos esenciales sin interrumpir el flujo de trabajo.

    Secretos

    Esta función proporcionará una lista desplegable de variables definidas como secretos en la pestaña de configuración. Este código variará según su instancia de ensayo.

    Variables

    Esta función proporcionará una lista desplegable de variables definidas previamente que el usuario puede llamar con un clic de botón.

    Variables dinámicas

    Esta función proporcionará una lista desplegable de variables definidas por el sistema que se pueden agregar al flujo de mAIstro. El usuario debe evaluar la plantilla primero antes de poder usar esta función.

    Nuevo

    Esta función generará un nuevo indicador de variable: << name: myVar, prompt: true >>.

    Apilar elementos

    Agregar nodos, de forma predeterminada, conectará los elementos verticalmente. Llamamos a esto Apilar o construir un Flujo.

    Los elementos apilados fluyen de arriba hacia abajo, lo que significa que la salida producida por el elemento superior estará disponible como entrada para el elemento inferior/siguiente.

    Encadenar elementos

    También puede conectar los elementos horizontalmente. Esto se llama Encadenar.

    El encadenamiento es útil cuando desea dirigir la salida de un nodo. En este ejemplo, la salida del LLM se proporcionará como entrada al elemento de extracción de palabras clave: encadenado juntos.

    Ejemplo:

    1. Haga clic en el elemento Extraer palabras clave para que se apile debajo de Enviar a LLM.
    2. Seleccione el nodo y arrástrelo al lado derecho del elemento que desea encadenar. Verá un punto azul que indica la conexión encadenada.

    3. Suelte la selección, encadenando los nodos juntos.

    Evaluación

    Hacer clic en el botón de evaluación ejecutará la expresión y generará la salida.

    Guardar como plantilla de usuario

    Es posible que use con frecuencia la misma expresión una y otra vez. Ofrecemos la posibilidad de guardar la plantilla para volver a usarla y también activarla a través de una llamada API.

    Construya una expresión y luego haga clic en el botón Guardar en la parte inferior del editor. Ingrese el nombre de la plantilla y la descripción (opcional). Haga clic en Guardar en el cuadro de diálogo para guardarla como una plantilla de usuario.

    Cargar la plantilla

    Su plantilla guardada se puede cargar en el editor o llamar más tarde desde la API.

    Haga clic en el botón Cargar en la parte inferior del editor, seleccione Plantillas de usuario y marque la casilla de la plantilla que desea cargar. Haga clic en Cargar plantilla para cargar la plantilla guardada en el editor.

    Formatos de salida

    • En línea - Adecuado para mostrar la salida renderizada del flujo, compatible con gráficos y HTML.

    Ejemplo: Mostrar un gráfico usando datos recuperados de un punto final de API, renderizado con Chart.js.
