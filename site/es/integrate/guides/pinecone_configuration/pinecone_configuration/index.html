    Integración de Pinecone con NeuralSeek

    Esta guía proporciona instrucciones paso a paso para configurar Pinecone como la base de conocimiento y usarlo junto con el modelo de incrustación. Además, se proporciona una explicación técnica de cómo funciona esta configuración. También se incluye un ejemplo de un script de Node.js para cargar documentos en el índice de Pinecone.

    Si bien esta guía se centra en Pinecone, vale la pena señalar que también puede usar Milvus como una base de datos de vectores alternativa.

    Requisitos previos

    • Asegúrese de tener Node.js instalado.

    Pasos

    1. Crear una cuenta de Pinecone

    • Vaya a Pinecone y cree una nueva cuenta.

    2. Crear un nuevo índice en Pinecone

    • Navegue al panel de control y cree un nuevo índice.

    • Dependiendo del modelo de incrustación que planee usar, elija el tamaño de vector apropiado:

      • text-embedding-ada-002: Tamaño de vector 1536
      • text-embedding-3-small: Tamaño de vector 1536
      • text-embedding-3-large: Tamaño de vector 3072
      • infloat-e5-small-v2: Tamaño de vector 384

    3. Configurar NeuralSeek

    3.1. Configurar la conexión de la base de conocimiento

    • Acceda a la plataforma NeuralSeek.
    • Vaya a la pestaña Configurar y configure la conexión de la base de conocimiento:
    • Tipo de base de conocimiento: Pinecone
    • Idioma de la base de conocimiento: Inglés
    • Nombre del índice de Pinecone: docs
    • Espacio de nombres del índice de Pinecone: ns1
    • Clave de API de Pinecone: your-pinecone-api-key
    • Campo de datos de curaduría: text
    • Campo de nombre de documento: title
    • Campo de filtro: title
    • Campo de enlace: link
    • Recursos de atributos: habilitado

    3.2. Agregar un modelo de incrustación

    • Vaya a la sección Modelos de incrustación y agregue una nueva incrustación:

    • Elija la plataforma (ya sea Azure, NeuralSeek u OpenAI).

    • Seleccione el modelo de incrustación apropiado:
      • Para OpenAI y Azure:
      • text-embedding-ada-002: Tamaño de vector 1536
      • text-embedding-3-small: Tamaño de vector 1536
      • text-embedding-3-large: Tamaño de vector 3072
      • Para NeuralSeek:
      • infloat-e5-small-v2

    Incrustaciones de texto

    • Convierte los datos de texto en representaciones vectoriales densas que capturan información semántica.

    Coincidencia de similitud

    • Compara los vectores de consulta con los vectores de documentos para encontrar las respuestas más relevantes.

    Comprensión contextual

    • Aprovecha múltiples capas para entender y generar respuestas precisas en el contexto.

    Flujo de trabajo de integración

    1. Ingesta de datos: Los documentos se ingresan y procesan para generar incrustaciones vectoriales utilizando el modelo de incrustación de NeuralSeek.
    2. Indexación: Las incrustaciones vectoriales generadas se almacenan en Pinecone, donde se indexan para una búsqueda y recuperación eficientes.
    3. Procesamiento de consultas: Cuando se ingresa una consulta, NeuralSeek convierte el texto de la consulta en un vector utilizando el modelo de incrustación.
    4. Búsqueda y recuperación: El vector de consulta se compara con los vectores de documentos en Pinecone para encontrar las coincidencias más relevantes.
    5. Generación de respuestas: Se recuperan los documentos más relevantes de Pinecone, y NeuralSeek formula una respuesta en función de los datos recuperados.

    Beneficios de esta configuración

    Eficiencia

    • Combinar las capacidades eficientes de búsqueda vectorial de Pinecone con las poderosas incrustaciones de NeuralSeek asegura respuestas rápidas y precisas.

    Escalabilidad

    • Pinecone puede escalarse para manejar grandes volúmenes de datos, mientras que las incrustaciones de NeuralSeek mantienen un alto rendimiento.

    Precisión

    • Las incrustaciones contextuales de NeuralSeek mejoran la precisión de las respuestas, proporcionando información relevante y precisa.

    Solución de problemas

    Problema: El modelo no proporciona respuestas precisas

    • Solución: Verificar los parámetros del modelo y asegurarse de que el contenido de la base de conocimientos esté actualizado.

    Problema: Errores de carga

    • Solución: Asegurarse de que los formatos de archivo sean correctos y que la integridad de los datos se mantenga.

    Problema: Problemas de integración

    • Solución: Volver a verificar el vínculo entre el modelo y la base de conocimientos, y verificar que la sincronización esté configurada correctamente.
