    Editor Visual

    Visão geral

    Apresentando o mAIstro - um playground aberto para Modelos de Linguagem Grandes, projetado para facilitar o tempo e o esforço de desenvolvimento.

    O mAIstro é uma ferramenta prática que lhe fornece as seguintes capacidades:

    1. Escolha de LLM: (Planos BYOLLM) Selecione seu LLM preferido e integre-o perfeitamente com o mAIstro.
    2. Utilize a Linguagem de Modelo NeuralSeek (NTL): Crie prompts dinâmicos usando uma combinação de palavras regulares e marcação NTL para recuperar conteúdo de diferentes fontes.
    3. Editor Visual Amigável: Crie prompts personalizados com um editor visual fácil de usar, basta apontar e clicar.
    4. Utilize Outros Recursos NeuralSeek: Extraia, proteja ou procure uma consulta através da plataforma mAIstro.
    5. Recuperação de Conteúdo Versátil: Recupere dados de várias fontes, incluindo Bases de Conhecimento, Bancos de Dados SQL, sites, arquivos locais ou seu próprio texto.
    6. Melhoria de Conteúdo: Melhore seus dados com recursos como resumo, remoção de stopwords, extração de palavras-chave e remoção de PII para garantir que seu conteúdo seja refinado e valioso.
    7. Prompts Protegidos: O mAIstro fornece Proteção contra Injeção de Prompt e Barreiras de Profanidade, evitando momentos embaraçosos com a Geração de Linguagem.
    8. Compreensão de Tabelas: Realize pesquisas e gere respostas com consultas em linguagem natural contra dados estruturados.
    9. Saída Sem Esforço: Visualize facilmente seu conteúdo gerado no editor integrado ou exporte-o diretamente para um documento do Word, oferecendo controle conveniente sobre sua saída.
    10. Pontuação Semântica de Precisão: Importante, todas essas operações são avaliadas usando nosso modelo de Pontuação Semântica. Isso permite obter insights sobre o escopo do conteúdo, adaptado às suas preferências.

    Editor Visual

    • O Editor Visual permite que os usuários criem expressões usando blocos móveis, encadeados e personalizáveis que executam comandos. Ele simplifica a interação do usuário por meio de blocos de arrastar e soltar, facilitando a navegação em casos de uso complexos sem a necessidade de código.

    Editor NTL

    • O Editor NTL permite que usuários avançados ou desenvolvedores criem expressões usando a Marcação NTL. Isso mostra o Markdown NTL bruto, permitindo que você edite manualmente ou copie todo o modelo para compartilhar e depurar.

    Inspetor do mAIstro

    • O Inspetor do mAIstro (o pequeno ícone de inseto próximo ao canto superior direito) permite que os usuários aprofundem os detalhes de cada etapa, expondo o que foi definido, quando foi definido e como foi processado.
    • Expanda as etapas individualmente para aprofundar em valores, cálculos, atribuições ou geração específicos.

    Início rápido com construtor automático

    Comece dando um prompt no construtor automático. Use este prompt de exemplo: Construa um modelo para enviar e-mails individualizados para cada endereço listado em um arquivo CSV de entrada. Isso lhe dá a capacidade de começar do zero, usar um modelo existente ou construir um usando comandos em linguagem natural.

    Isso irá gerar um modelo personalizável que você pode testar ou adaptar às suas necessidades.

    Entendendo o Editor Visual

    Clique para inserir

    Todos os elementos no painel esquerdo podem ser criados no editor clicando neles.

    Clique para editar

    Selecionar um cartão destacará o nó em azul e uma caixa de diálogo aparecerá no lado direito para editar as opções de configuração do nó selecionado. Dependendo do tipo do nó, pode haver várias opções. Consulte a página de referência NTL para obter uma descrição de todas as opções configuráveis.

    Excluindo um nó

    Você pode excluir um nó clicando no botão vermelho Excluir nó na parte inferior do painel de opções.

    Menus de Hover

    Os menus de hover permitem que os usuários acessem e insiram facilmente segredos, variáveis ​​definidas pelo usuário, variáveis ​​definidas pelo sistema ou gerem novas variáveis ​​enquanto trabalham no construtor visual. Esse recurso melhora o processo de construção, fornecendo acesso rápido a elementos essenciais sem interromper o fluxo de trabalho.

    Segredos

    Esta função fornecerá um menu suspenso de variáveis ​​definidas como segredos na guia de configuração. Este código vai variar dependendo da sua instância de teste.

    Variáveis

    Esta função fornecerá uma lista suspensa de variáveis ​​previamente definidas que o usuário pode chamar com um clique de botão.

    Variáveis ​​Dinâmicas

    Esta função fornecerá uma lista suspensa de variáveis ​​definidas pelo sistema que podem ser adicionadas ao fluxo mAIstro. O usuário deve avaliar o modelo primeiro antes de poder usar este recurso.

    Novo

    Esta função gerará um novo prompt de variável: << name: myVar, prompt: true >>.

    Empilhando elementos

    Adicionar nós, por padrão, conectará os elementos verticalmente. Chamamos isso de Empilhamento ou construção de um Fluxo.

    Os elementos empilhados fluem de cima para baixo, o que significa que a saída produzida pelo elemento superior ficará disponível como entrada para o elemento inferior/próximo.

    Encadeando elementos

    Você também pode conectar elementos horizontalmente. Isso é chamado de Encadeamento.

    O encadeamento é útil quando você deseja direcionar a saída de um nó. Neste exemplo, a saída do LLM será fornecida como entrada para o elemento de extração de palavras-chave - encadeado juntos.

    Exemplo:

    1. Clique no elemento Extrair palavras-chave para empilhar abaixo de Enviar para LLM.
    2. Selecione o nó e arraste-o para o lado direito do elemento que você deseja encadear. Você verá um ponto azul indicando a conexão encadeada.

    3. Solte a seleção, encadeando os nós juntos.

    Avaliando

    Clicar no botão de avaliação executará a expressão e gerará a saída.

    Salvando como modelo de usuário

    Você pode usar a mesma expressão com frequência. Oferecemos a capacidade de salvar o modelo para reutilização e também ser acionado via uma chamada de API.

    Construa uma expressão e, em seguida, clique no botão Salvar na parte inferior do editor. Digite o nome do modelo e a descrição (opcional). Clique em Salvar na caixa de diálogo para salvá-lo como um modelo de usuário.

    Carregando o modelo

    Seu modelo salvo pode ser carregado no editor ou chamado posteriormente da API.

    Clique no botão Carregar na parte inferior do editor, selecione Modelos de Usuário e marque a caixa de seleção do modelo que você deseja carregar. Clique em Carregar Modelo para carregar o modelo salvo no editor.

    Formatos de saída

    • Inline - Adequado para exibir a saída renderizada do fluxo, com suporte a gráficos e HTML.

    Exemplo: Exibir um gráfico usando dados recuperados de um endpoint de API, renderizado com Chart.js.
