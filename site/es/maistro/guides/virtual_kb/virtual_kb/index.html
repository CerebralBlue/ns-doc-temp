    Base de conocimiento virtual

    Descripción general

    ¿Qué es?

    • Virtual KB es una función en mAIstro que le permite definir un flujo y usarlo como una base de conocimiento virtual. Esta función le permite combinar múltiples fuentes de conocimiento en una sola base de conocimiento unificada, proporcionando una solución más completa y flexible para sus necesidades de recuperación de información.

    ¿Por qué es importante?

    • Una Virtual KB mejora la búsqueda y el descubrimiento de su aplicación al integrar múltiples fuentes de conocimiento, entregando resultados más completos y relevantes. Ofrece flexibilidad y escalabilidad, lo que le permite ajustar fácilmente las fuentes de conocimiento a medida que cambian sus necesidades.

    ¿Cómo funciona?

    • Virtual KB le permite conectar e integrar varias fuentes de conocimiento, como bases de datos, sistemas de gestión de contenido y API externas, en una sola base de conocimiento virtual. Comience construyendo un flujo en mAIstro utilizando nuestra variedad de funciones y conectores nativos o consulte nuestra plantilla de ejemplo de Virtual KB para una guía fácil sobre cómo configurar una Virtual KB.

    Plantilla de ejemplo en mAIstro

    1. Navegue a la pestaña mAIstro en su instancia de NeuralSeek.
    2. Haga clic en Plantillas de ejemplo y busque la plantilla titulada Virtual KB.

    Este flujo utiliza los nodos Virtual In y Virtual Out, ubicados debajo de RAG Tools en el menú lateral. Pasa un conector de búsqueda de DuckDuckGo y un conector de API Rest con una URL de Wikipedia al Modelo de Lenguaje Grande para la generación de respuestas dentro de la pestaña Seek. Ahora podemos utilizar la World Wide Web como fuente de conocimiento para la generación de respuestas.

    Seleccionando una Virtual KB

    1. Navegue a la pestaña Configurar en su instancia de NeuralSeek.
    2. Expanda el acordeón Conexión de Base de Conocimiento.
    3. Para el Tipo de Base de Conocimiento, seleccione la opción Virtual KB.
    4. Para la plantilla mAIstro Virtual KB, seleccione la opción ex_Virtual_KB.
    5. Haga clic en el icono rojo Guardar en la parte inferior de la pantalla para guardar su configuración.

    Buscar con un teclado virtual

    1. Navega a la pestaña Buscar en tu instancia de NeuralSeek.
    2. Escribe cualquier pregunta. Por ejemplo, ¿Quién es Taylor Swift?
    3. Haz clic en el botón Buscar para generar una respuesta.

    Mientras revisamos la respuesta generada, podemos resaltar los detalles estadísticos y la fuente devuelta por NeuralSeek. La respuesta se sintetiza a partir de una combinación de búsquedas de DuckDuckGo y Wikipedia relacionadas con la cantante. Nuestro análisis semántico nos dice sobre los diversos saltos entre los artículos fuente. Teniendo en cuenta que hay una gran cantidad de información en Wikipedia sobre Taylor Swift, también recibimos una puntuación de cobertura de KB del 99%.

    Al expandir las fuentes a continuación, podemos examinar cada una en detalle. Los indicadores de procedencia resaltan las palabras clave y frases extraídas de cada fuente para formar la respuesta final.

    Expandir tu base de conocimiento

    En última instancia, puedes conectar prácticamente cualquier fuente de conocimiento a tu instancia de NeuralSeek para la generación de respuestas a través de los conectores de Virtual KB en mAIstro. Puedes elegir entre una variedad de conectores de bases de datos integrados, conectores de bases de conocimiento o conectores de búsqueda web. O bien, conéctate a cualquier fuente adicional a través de nuestro nodo conector de API Rest.

    Construir un flujo

    1. Navega a mAIstro en tu instancia de NeuralSeek.
    2. Selecciona el nodo Virtual KB - In del menú lateral bajo RAG Tools.

    Este nodo te da varias variables para usar dentro de tu flujo.

    1. Selecciona el nodo Website Data del menú lateral bajo Get Data. Esto se vinculará automáticamente debajo de tu primer nodo.
    2. Haz clic en el icono de engranaje para ingresar cualquier URL válida. En este ejemplo, nos estamos conectando a una búsqueda de Google: https://www.google.com/search?gfns=1&q=<< name: virtualKbIn.contextQuery>>
    3. Selecciona el nodo Set Variable del menú lateral bajo Control Flow.
    4. Haz clic y arrastra el nodo Set Variable a la derecha del nodo Website Data para encadenarlo.
    5. Haz clic en el icono de engranaje para establecer el nombre de la variable. En este ejemplo, el nombre de la variable es google.

    La adición de la variable virtualKbIn.contextQuery permite que el contexto de la consulta del usuario se transfiera dinámicamente a la búsqueda de Google.

    1. Selecciona un segundo nodo Website Data.
    2. Haz clic en el icono de engranaje para ingresar cualquier URL adicional. En este ejemplo, nos estamos conectando a la página de documentación de NeuralSeek: https://documentation.neuralseek.com/
    3. Selecciona el nodo Set Variable del menú lateral bajo Control Flow.
    4. Haz clic y arrastra el nodo Set Variable a la derecha del segundo nodo Website Data para encadenarlo.
    5. Haz clic en el icono de engranaje para establecer el nombre de la variable. En este ejemplo, el nombre de la variable es docs.

    Hemos agregado la documentación de NeuralSeek como una segunda fuente de referencia para nuestra base de conocimiento y estamos realizando una extracción estática de la información del sitio web.

    1. Selecciona el nodo Virtual KB - Out del menú lateral bajo RAG Tools.
    2. Haz clic en el icono de engranaje para configurar la información que se enviará de vuelta a Seek. En este ejemplo, queremos definir el pasaje incluyendo los nombres de las variables: << name: google >>\n<< name: docs >>.
    3. Además, podemos preestablecer el kbCoverage, kbScore, url y el nombre del documento. En este ejemplo, definimos el nombre del documento como Virtual KB.
    4. Guarda tu flujo de mAIstro con un nombre único y una descripción opcional. En este ejemplo, el nombre es websiteKB. Ahora se extraerán en vivo los dos sitios web cada vez que llegue una solicitud. La información extraída de los sitios se generará de forma dinámica y en paralelo, y luego se volverá a insertar en el proceso de solicitud para generar la respuesta.

    Nota

    Si bien aquí utilizamos un solo documento concatenado por simplicidad, es posible dividirlo en varios documentos. Simplemente cree un objeto JSON con una matriz de objetos de documento que contengan las propiedades: documento (título), url, puntuación y pasaje.

    {{ virtualKbIn  }}
{{ web  | url: https://www.google.com/search?gfns=1&q=<< name: virtualKbIn.contextQuery>> }}=>{{ variable  | name: google }}
{{ web  | url: https://documentation.neuralseek.com/ }}=>{{ variable  | name: docs }}
{{ virtualKbOut  | context: << name: google >>\n<< name: docs >> | kbCoverage: 0 | kbScore: 0 | url:  | document: Virtual KB }}

    Configuración de un Virtual KB

    1. Navegue a la pestaña Configurar en su instancia de NeuralSeek.
    2. Expanda el acordeón Conexión de la base de conocimiento.
    3. Para el tipo de base de conocimiento, seleccione la opción Virtual KB.
    4. Para la plantilla mAIstro Virtual KB, seleccione la opción websiteKB.
    5. Haga clic en el icono rojo Guardar en la parte inferior de la pantalla para guardar su configuración.

    Buscar con un Virtual KB

    1. Navegue a la pestaña Buscar en su instancia de NeuralSeek.
    2. Escriba cualquier pregunta. Por ejemplo, ¿NeuralSeek proporciona un Hands-On Lab?
    3. Haga clic en el botón Buscar para generar una respuesta.

    Podemos expandir la fuente de Virtual KB debajo del contexto de la base de conocimiento y ver qué información se extrajo de la búsqueda de Google y qué se extrajo de nuestra URL de documentación de NeuralSeek para generar la respuesta.
