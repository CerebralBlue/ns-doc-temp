    Changelog

    Octubre - 2024

    Nuevas características

    • Actualizaciones de mAIstro:
      • NTL ahora tiene resaltado de código y rollup y un nuevo editor
      • Caja de herramientas de código: un conjunto de funciones fáciles de un solo nodo para:
      • extraer código generado / sql / html de la mayoría de los LLM,
      • proteger, validar y reescribir SQL
      • Limpiar HTML y extraer texto
    • Actualizaciones de HTML Cleanser. NeuralSeek limpia automáticamente los documentos HTML raspados en KB's a los que te conectas. Ahora puedes especificar selectores CSS para eliminar además de nuestra limpieza normal, así como deshabilitar el limpiador.
    • Gobernanza - Información sobre costos. Ambos lados de la gobernanza obtienen una nueva pestaña que compara el costo de los modelos seleccionados con todos los demás modelos para los que tenemos datos de capacidad y costo.
    • DQL para elastic / watsonx discovery. Hemos llevado nuestro intérprete DQL a elastic para que puedas pasar filtros DQL y hacer fácilmente filtrado complicado y reducir los riesgos de migración cuando vengas de discovery.

    Septiembre - 2024

    Nuevas características

    • Constructor visual de agentes múltiples. Activa el modo de agentes múltiples en la pestaña de configuración y construye fácilmente flujos de agentes múltiples impulsados por categorías. ¡No se requiere codificación! Cada nodo del árbol de agentes múltiples puede tener su propia configuración (kb, llm's, todo) y guardrailings. También hemos unificado los lados de Seek y mAIstro de la casa para que puedas llamar a ambos desde cualquier api.
      • Cada nodo puede ser de un tipo de búsqueda tradicional o de un nuevo nodo liderado por mAIstro. Los nodos de mAIstro envían intenciones que los golpean a una acción predeterminada en lugar de crear una nueva intención. Esto te permite hacer desambiguación o enfocar al usuario en las capacidades que has habilitado, como abrir tickets de problemas u otras acciones lideradas por mAIstro.
      • Cualquier intención puede ejecutar directamente un flujo de mAIstro. Entonces, para una pregunta como ¿cómo está el clima hoy?, podrías llamar a un flujo de mAIstro en lugar de enviarlo al camino/kb de búsqueda tradicional.
      • Ahora puedes agregar una nueva intención directamente a través de la pestaña de configuración y curación.
      • Los guardrailings pueden ejecutar flujos de mAIstro. La confianza mínima, las palabras mínimas y máximas pueden ejecutar flujos de mAIstro personalizados para dar respuestas específicas del contexto y el idioma cuando se activen esos guardrailings.
    • ¡Chat! Hemos introducido una interfaz similar a ChatGPT, así como un SDK y código de incrustación. Puedes agregar rápidamente un agente virtual a tu sitio web simplemente colocando el código de incrustación. El SDK de Chat permite operaciones de arrastrar y soltar de imágenes y archivos, por lo que podrías construir fácilmente un bot que permita a los usuarios hacer preguntas proporcionando imágenes, como quiero un refrigerador como este.
    • ¡OCR! Lanzamos en silencio nuestras capacidades de OCR hace unas semanas, pero nunca lo anunciamos. Ahora tenemos OCR integrado en el sistema. Cuando uses o cargues un documento en mAIstro, automáticamente OCR cualquier PDF que encontremos que esté basado en imágenes y no en texto. También puedes OCR archivos de imagen. Además, lanzamos una plantilla de doble OCR, básicamente mostrándote cómo aprovechar y paralelizar nuestro OCR además de las capacidades visuales de un modelo multimodal para hacer cosas increíbles para OCR de documentos complejos mientras se mantiene el formato de origen. Esta capacidad supera con creces a las herramientas de OCR heredadas, con casi ningún ajuste.
    • Generación de documentos: hemos construido una nueva generación de documentos detrás de escena, lo que permite una mayor capacidad para generar archivos PDF y Word bien formateados a escala.
    • ¡LLM's! LLama3.2 tanto en bedrock como en watsonx.ai. Este es el primer modelo multimodal disponible en watsonx.
    • ¡Nuevo modelo curado! Para los planes PPA, el modelo curado 1.1 está disponible en todas las regiones.
    • Actualizaciones de registro. Los registros de Seek se han trasladado a la pestaña de Gobernanza e incluyen aún más detalles. Habilitar el registro corporativo para la reproducción de transacciones, que se ha convertido rápidamente en un elemento imprescindible para las pymes.
    • ¡nuevos nodos mAIstro! Lanzamos un kit de herramientas XML, así como una docena de otros nodos y conectores nuevos.
    • ¡mejoras de traducción! A través de la API, puede anular el tamaño máximo de fragmento que usamos, lo que puede acelerar dramáticamente las traducciones para traducciones de longitud media cuando se utilizan LLM / plataformas de inferencia lentas.

    Agosto - 2024

    Nuevas características

    • mAIstro Confianza mínima - En una búsqueda, al alcanzar el umbral de confianza mínima, ejecute un flujo personalizado de mAIstro. Puede usar esto simplemente para crear una nota de "no sé" consciente del contexto, pero también puede usarlo para iniciar una notificación, una escalación o una llamada a un servicio externo o un ticket... Cualquier cosa realmente.
    • Insights semánticos - ahora en el gráfico de términos alucinados, puede hacer clic en un término para permitir enumerar elementos directamente...
    • Cargador de datos: arrastre y suelte archivos a nuestro nuevo cargador, que aprovecha mAIstro para dividir y cargar documentos. Puede usar cualquier función o integración de mAIstro, hacer llamadas REST, generar incrustaciones, recorrer y dividir automáticamente documentos... Damos un ejemplo de cargador para elastic/watsonx discovery.
    • ¡Gobernanza para mAIstro!
      • Realice un seguimiento y proporcione información sobre todas las plantillas de mAIstro, filtrable por plantilla.
      • Flow insights ayuda a realizar un seguimiento del tiempo dedicado en nuestro motor paralelo, lo que le ayuda a optimizar los flujos y a comprender dónde pasan la mayor parte de su tiempo.
      • Token insights refleja la pestaña de información de tokens de Seek, lo que ayuda a mostrar el consumo de tokens, el costo y las opciones de comparación de modelos para los LLM utilizados para alimentar sus flujos de mAIstro.
    • Actualizaciones de gobernanza de Seek
      • Filtrar por filtro... Cuando use filtros en Seek, ahora puede realizar un seguimiento automático de la gobernanza por el filtro aplicado.

    Julio - 2024

    Nuevas características

    • Nuevos LLM
      • Mistral-large en watsonx.ai
      • GPT-4o-mini en OpenAI.
    • Endpoints de API de transmisión en vivo para Seek y mAIstro para el asistente de watsonx. Estos tienen el tipo de contenido requerido en la especificación de OpenAPI. Nota: por el momento, no se recomienda la búsqueda en vivo, ya que no se puede usar la puntuación de confianza ni obtener campos de carga útil como la URL. Trabajaremos en esto con el equipo de Watson. Nuevos modelos de incrustación y la capacidad de usar un modelo de incrustación personalizado con detección de intención de NS y mAIstro
    • ¡Mejoras en la traducción! La traducción de NS ahora es hasta un 80% más rápida para traducciones grandes.
    • LLM alojados de NeuralSeek. Cuando use un plan BYO-LLM, ahora proporcionamos un LLM base alojado globalmente (mistral-7b) y un LLM de traducción diseñado específicamente para ese plan sin cargos adicionales, solo se aplica el cargo normal de Seek. Esto debería facilitar mucho el inicio con NS.

    Junio - 2024

    Nuevas características

    • Nuevas plataformas compatibles:
      • vLLM / motores de inferencia genéricos estilo OpenAI. Esto le permite enchufar y reproducir con muchos más motores de inferencia on-prem y SaaS.
      • Google Vertex ahora es compatible, y hemos agregado Gemini 1.5 pro y Flash. Estos modelos son bastante buenos: Pro está en el mismo nivel que GPT-4o, Claude3 Sonnet y Mistral-Large.
    • ¡Actualizaciones de mAIstro!
      • Gráficos integrados. Con un LLM compatible, puede solicitar que se genere un gráfico como parte de la salida.
      • Salida con formato: genere y muestre HTML y JavaScript.
      • Nueva vista Raw: vea el código detrás de los gráficos y el HTML generado.
      • Salida PDF
      • ¡Menús emergentes! Cuando esté en el constructor visual, todos los nodos ahora le permitirán ver e insertar cualquier secreto, variable definida por el usuario y el sistema, o generar una nueva variable. ¡Hace que la construcción sea mucho más fácil!

    • Integración nativa con watsonx.governance. En mAistro, vea nuestra plantilla de ejemplo sobre cómo configurar esto, es realmente fácil, solo 3 pasos. Para watsonx.governance, solo necesita una clave de API de IAM y, desde su espacio de implementación de producción en x.gov, en acciones/información del modelo, necesitamos su ID de centro de datos de evaluación e ID de suscripción. Enviaremos todas las medidas de NeuralSeek a watsonx.governance para que pueda recopilar y gobernarlas entre instancias y mostrar una historia de gobernanza más amplia. También proporcionamos una integración abierta en caso de que desee hacer algo más personalizado.

    • Nuevas integraciones de mAIstro: (¡hay tantas funciones nativas y conectores en mAistro que tuvimos que agregar una función de búsqueda!)

      • Jira
      • Trello
      • Github
      • Slack
      • AWS S3
      • Búsquedas web de Google/Bing/Yahoo/DuckDuckGo.

    • Herramientas JSON: Agregamos el filtro de matriz JSON y la función de escape JSON para facilitar el trabajo con cargas útiles complicadas dentro de mAIstro.

    • Escape automático. Ahora, al usar el editor visual de mAIstro, escaparemos automáticamente cualquier comilla. Esto debería facilitar la construcción en mAIstro para los usuarios empresariales. Hemos descubierto que estas actualizaciones, junto con el generador automático de mAIstro que lanzamos el mes pasado, reducen muchos casos de uso a trabajar sin problemas sin modificaciones adicionales requeridas en los flujos generados automáticamente.

    • Actualizaciones de gobernanza: Hemos mejorado la pestaña Información del token y agregado un nuevo gráfico Resolución de preguntas a la pestaña Descripción general para ayudar a rastrear cuántas preguntas alcanzan el umbral de confianza mínimo.

    • La pestaña Registros ahora marca las respuestas que tenían acciones de PII, activación de HAP e inyección de indicaciones.

    Mayo - 2024

    Nuevas características

    • ¡KB virtuales! Ahora puede usar mAIstro para definir un flujo y usarlo como una base de conocimiento virtual. ¿Quiere consultar varias instancias de descubrimiento a la vez? Fácil. ¿Elastic y DB2 y fusionar los resultados? Fácil. ¿Raspar algunas páginas web en vivo y usar esas? Fácil. Consulte la nueva plantilla en mAIstro para ver un ejemplo de cómo configurar esto.

    • Lista de permitidos semántica (Configuración / Ajuste del modelo semántico). Especifique palabras o frases que se deben excluir de las penalizaciones semánticas.

    • Actualizaciones de curación. Ahora las respuestas generadas por el uso de un filtro mostrarán el filtro utilizado durante la generación.

    • Traducciones personalizadas. Cargue un archivo de entrenamiento a través de la API.

    Características de mAIstro

    • Procesamiento de imágenes / soporte multimodal en mAIstro. Ahora puede obtener imágenes de la web, un archivo local o Google Docs y fluirlas a través de LLM que admiten el procesamiento de imágenes (Claude3, GPT-4, GPT-4o). Consulte la nueva plantilla de ejemplo. ¡Y sí, puede alimentar Seek en función de imágenes si usa esto con la KB virtual!

    • Generador automático para mAIstro (solo SaaS). ¿Ha estado abrumado o temeroso de probar mAIstro? ¿No está claro cómo construir algo? Ahora el modal de bienvenida (y el modal de carga) le pedirá que simplemente describa su caso de uso y luego le generaremos una plantilla personalizada.

    • ¡Conector de Snowflake! Ahora disponible en mAIstro.

    Características de gobernanza

    • ¡Información del token! Un nuevo módulo llega a NeuralSeek Governance (solo planes BYO-LLM). Obtenga información sobre los costos de su uso de LLM, métricas sobre la velocidad de generación, comparaciones de costos con LLM de capacidad similar. Es muy convincente.

    • Actualizaciones de gobernanza: ahora puede rastrear el porcentaje de aciertos en caché y respuestas editadas desde la pestaña Información semántica.

    Nuevos modelos

    • Muchos nuevos. GPT-4o, Mixtral8x-22 y más.

    Abril - 2024

    El lanzamiento de NeuralSeek Governance.

    Nuevas características

    • Eliminar alucinaciones: actívelo a través de la pestaña Configurar en Puntuación semántica. Como parte de una respuesta de Seek, elimine cualquier oración que contenga una palabra clave (nombre propio, entidad) que no esté contenida en su documentación de origen.

    • Propuestas. Nuestra visión sobre el control de versiones / cambios de configuración. Ahora puede definir una configuración como una Propuesta y luego llamar a esa propuesta dinámicamente desde la API o las pestañas Seek o Home. Esto ayuda a separar la configuración administrativa de las pruebas de cambios propuestos por los expertos. También le permite ejecutar varias configuraciones a la vez sin pasar una anulación completa cada vez. Actualice una configuración y haga clic en Proponer cambios. Además, una nueva función Registrar configuraciones alternativas le permite bloquear la curación de respuestas provenientes de estas propuestas, para que pueda probar de forma aislada en una sola instancia. Título y descripción de la configuración: como parte de nuestro módulo de Gobernanza y el lanzamiento de propuestas, ahora le pediremos un título y una descripción de la configuración al guardar. Estos fluyen hacia el lado de la gobernanza de la casa para la explicabilidad.

    • Soporte de Pinecone: nuestro lanzamiento inicial. Más opciones de modelos de incrustación llegarán en breve.
    • Conector de KB de Milvus. Ahora puede realizar búsquedas vectoriales en watsonx.data
    • Devolver documentos completos: estamos implementando la capacidad de devolver un documento completo en lugar de un pasaje. Actualmente lanzado para Discovery y AppSearch. De esta manera, si ha creado o pre-recortado cuidadosamente su documentación, puede asegurarse de que se devuelva el documento completo.
    • Mejoras de rendimiento: algunas actualizaciones importantes en áreas como el raspado web dinámico, la división de la ventana de contexto y más.

    Características de mAIstro

    • ¡Secretos! - defina variables en la pestaña Configurar para ocultarlas de los usuarios normales de mAIstro. Los usuarios on-premise también pueden definir variables a nivel del sistema operativo. Muy útil para pasar / ocultar información de conexión a la base de datos.
    • Bucle de contexto: divida un bloque de texto grande por tokens y recorra. Muy útil para traducir documentos grandes o enviar cosas grandes a través de un LLM pequeño. Consulte el ejemplo de Traducción de documentos en mAIstro
    • Conector de Google Drive: extraer y escribir en Google Drive
    • Bucle de variables: recorrer una matriz de datos

    Características de gobernanza

    • Módulo de gobernanza. Nuestro enfoque inicial con este primer lanzamiento es una visión holística de la gobernanza RAG con filtrado basado en el tiempo y la intención/categoría. Estaremos implementando muchas más capacidades adicionales en las próximas semanas aquí. Al lanzamiento tenemos:
      • Gráficos de resumen ejecutivo
      • Análisis de intenciones: qué intenciones están en tendencia y cómo se están desempeñando: regresión de modelos / documentos
      • Rendimiento del sistema: monitoree su instancia y compárela con el universo NS
      • Información semántica: Cuál es la calidad de las respuestas generadas
      • Información de documentación: Qué documentación se usa más y cómo se está desempeñando
      • Información de configuración: monitorear los cambios de configuración y rastrear el desgaste a lo largo del tiempo

    Nuevos modelos

    • LLama 3: un gran paso adelante con respecto a llama 2 en términos de su capacidad para seguir instrucciones. En watsonx, la ventana de contexto es pequeña, sin embargo, mixtral sigue siendo mejor en general.
    • jais-13b-chat: en watsonx frankfurt, para casos de uso en árabe
    • granite-7b-lab: este parece mejor que los otros modelos de granito. Debajo de la superficie, se basa en llama-2...
    • Mistral-Large: similar y mejor de forma iterativa que mixtral. aún no disponible en watsonx.

    Marzo - 2024

    Explore ahora se llama mAIstro y ha ganado una variedad de nuevas funciones.

    Nuevas características

    • RAG totalmente personalizado ahora disponible en NeuralSeek, ofreciendo simplicidad a través de Seek y complejidad a través de mAIstro, todo listo y sin código.

    Características de mAIstro

    • Curar: envíe su propia P+R a las pestañas Curar, Analítica y Registro.
    • Categorizar: conéctese al clasificador NS para obtener categoría e intención.
    • Caché de consultas: busque y devuelva respuestas curadas y editadas.
    • Puntuación semántica: Accede al modelo de puntuación semántica desde dentro de un flujo de Maistro.
    • Extraer gramática: Extrae entidades, sustantivos, fechas y más del texto.
    • Agregar contexto: Recuerda el último turno de la conversación e inyecta el tema anterior en el texto (para una llamada a KB o LLM).
    • Detener: Detener la ejecución (útil para condicionales).
    • Truncar por tokens: Recortar el texto por un número determinado de tokens de LLM (usa esto para recortar tu documentación de KB para que quepa en la ventana de contexto de LLM).

    Nuevos modelos

    • Se han agregado dos nuevos modelos a watsonx en NeuralSeek: Granite 7B Japanese y Elyza Japanese Llama.

    Otras actualizaciones

    • Se ha agregado un nuevo paseo de introducción para ayudar a los nuevos usuarios a comenzar en mAIstro.

     

    Febrero - 2024

    Nuevas características

    • Filtrado/enmascaramiento de PII previo a LLM: Elimina o enmascarilla la información de identificación personal (PII) antes de enviar consultas a una base de conocimiento (KB) o LLM. Usa elementos preconfigurados o agrega los tuyos usando expresiones regulares.
    • Detección de inyección de indicaciones: La entrada del usuario se puntúa contra un modelo interno para identificar posibles intentos de inyección de indicaciones. Las palabras problemáticas se filtran y se puede bloquear toda la entrada en función de la probabilidad de inyección de indicaciones.
    • Traducción de KB multilingüe: Cuando se especifica un idioma de salida diferente al idioma de la KB, la entrada del usuario ahora se puede traducir automáticamente al idioma de la KB para obtener mejores respuestas.
    • Esquemas arbitrarios para Explore: NeuralSeek Explore ahora admite esquemas arbitrarios, lo que permite a los usuarios conectarlo a cualquier cosa que envíe una solicitud POST, procesarla y devolverla en el formato correcto. Esta función permite el reformateo dinámico de mensajes en función del contexto guardado, el historial de chat u otros criterios, brindando una experiencia más personalizada a los usuarios.
    • Actualizaciones a la mitigación de inyección de indicaciones: La función "pruébalo" ahora muestra las puntuaciones de diferentes frases elegibles para ser eliminadas de la entrada del usuario, mejorando las capacidades de detección de inyección de indicaciones.

    Nuevos modelos

    • watsonx.ai presenta el modelo Granite-20b-5lang-instruct-rc en vista previa técnica, y se agregan varios modelos nuevos a Bedrock.

    Mejoras en Explore

    • Ahora están disponibles en Explore salvaguardas como el filtro de blasfemias y la inyección de indicaciones.
    • Se han agregado varios nuevos ejemplos de plantillas para demostrar estas nuevas funciones.
    • Los usuarios ahora pueden modificar la plantilla de personalización de WA proporcionada en los ejemplos en la pestaña Explore para reformatear dinámicamente los mensajes que fluyen a través de Explore desde Watson Assistant, ofreciendo una experiencia de chatbot más personalizada.
    • Los parámetros de encabezado overrideschema y templatename en la API de Explore permiten una configuración y personalización sencillas de los esquemas en Explore, lo que permite una integración fluida con varios sistemas y aplicaciones.

     

    Enero - 2024

    Nuevas características

    • Se introdujeron trabajos de ejecución en paralelo en Explore, lo que permite una ejecución más rápida de plantillas complicadas, a menudo superando la codificación personalizada en Python.
    • Mejoras en la búsqueda de varios turnos: los usuarios ahora pueden controlar el número de turnos anteriores enviados al LLM para una experiencia más estilo ChatGPT.
    • Mejoras en Extracción:
    • Soporte para definir tipos de entidades de expresiones regulares y palabras clave, reduciendo la carga de trabajo en LLM más pequeños/menos capaces y mejorando la velocidad de extracción.

    Mejoras en Explore

    • Conectores directos a varias bases de datos, incluidas Postgres, Oracle, MySQL, MariaDB, MS SQL y Redshift.
    • Variables de sistema para inyectar fecha, hora, UUID, números aleatorios, etc.
    • Se agregó la funcionalidad 'Extraer' a Explore.
    • Generador de plantillas OpenAPI de Explore mejorado para una integración más sencilla con Watson Assistant.
    • Nuevas plantillas disponibles, incluidas Custom RAG, Causa de pérdida de seguro y Lógica condicional.
    • Opción de especificar el LLM a usar en los pasos de Explore LLM para evitar alcanzar los límites de velocidad y distribuir la carga de manera efectiva.

    Actualizaciones

    • Permisos de usuario más detallados: los usuarios ahora pueden otorgar acceso a pestañas mientras restringen la capacidad de escritura desde pestañas específicas.
